# 3D STD · Web 工作台

AI 生成 → 编辑摆盘 → 打印检查 → 导出 STL 的网页版 3D 打印创作工作台。
规格见 `docs/`(PRD v0.95 · 技术方案 v2.4 · Backlog v1.1)。当前进度:T1–T16 功能闭环 + M1.5.3 图生模型 + M1.6.3 AI 拆件建议 + M1.7 修复工作台 + M1.7.2 自交局部证据定位 + M1.7.3 连通壳拆件预览 + M1.7.4 平面切割候选只读预览。生产默认使用 AIHubMix Responses 兼容接口，官方 OpenAI 通道保留为回滚；当前优先把检查问题转化为可验证证据和可预览、可确认、可撤销的确定性操作，录屏与高度自主拆件后置。

渐进式拆件 Agent 的架构与产品方案见 [`docs/agent-split-architecture.md`](docs/agent-split-architecture.md)，评测计划见 [`docs/agent-eval-plan.md`](docs/agent-eval-plan.md)。M1.6.3 增加可定位/可检查的建议操作卡、耗时与 token 遥测、Gold Set 契约和自动评分；仍然只读，没有自动拆件能力。

## 首次上线(全程网页操作,无需命令行)

**A. 仓库准备**
1. 打开 github.com/bigcatdamao/3D-STD → Settings → General,拉到底部 Danger Zone → Change visibility → **Public**。
   (私有库协作方读不到;若坚持私有,改为每次把代码 zip 传进对话。)
2. 若仓库为空:点 Add file → Create new file,随便建一个 `init.txt` 提交,激活默认分支。

**B. 上传代码**
1. 解压本 zip。
2. 仓库页 Add file → **Upload files**,把解压出的**全部内容**(含 docs、src、worker 等文件夹)一次拖入,Commit。
   备选:打开 Codespace,把 zip 拖进左侧文件树,终端跑 `unzip *.zip && git add -A && git commit -m t1 && git push`。

**C. 绑定 Cloudflare(一次性)**
1. 注册/登录 dash.cloudflare.com(免费档即可)。
2. 左栏 Workers & Pages → Create → **Workers** → Import a repository → 授权 GitHub → 选 `3D-STD`。
3. 构建设置:Build command 填 `npm run build`,Deploy command 保持默认(`npx wrangler deploy`),保存。
4. 等云端构建完成,得到 `https://3d-std.<你的子域>.workers.dev`。

**D. T1 验收(打开上面的 URL)**
- 看到五区布局壳;
- 视口里绿色立方体「立」在打印床网格上(Z-up 正确);
- 顶栏右侧「服务层: 在线」(Worker 路由通)。
三条全过 = T1 完成,此后每次推送代码自动重新部署。

## T5 验收(世界 · 相机 · 选择)

打开线上地址,对照三条:

1. **世界(VIEW-01)**:深色打印床居中,10mm 细网格 + 40mm 主网格,前缘/右缘有「X/Y 256 mm」标注,床边界有半透明成型体积线框;右上角下拉可切 256/180/350/自定义,床与网格随之重算。
2. **鼠标与相机(VIEW-02/03)**:右键拖 = 旋转,中键拖 = 平移,滚轮 = 缩放,左键点对象 = 选中(琥珀描边);键 1/2/3/0 = 顶/前/侧/轴测,F = 聚焦选中,Home = 复位,5 = 透视/正交切换(切换前后画面大小不跳);左下角状态条的「历史」计数在任何相机操作后**不增加**。
3. **选择与死区(VIEW-02/04)**:按住对象拖 ≥3px = 移动(松手后历史 +1,Ctrl+Z 可撤销;拖动中按 Esc = 归位且历史不变);按下几乎不动就松开 = 仅选中、历史不变;空白处拖出框,擦到即选;灰色半透明的「已锁定 · 校准板」点不中、框不中,Ctrl+A 也不选它。

三条全过 = T5 关闭。

## T6 验收(Gizmo 与沉底)

前置:点选任一对象(琥珀描边出现后,视口中出现三色手柄;X 红 / Y 绿 / Z 蓝)。

1. **移动模式(VIEW-05)**:按 W(或工具栏「移动」)——三色箭头 + 三个半透明平面块。沿箭头拖 = 只沿该轴移动(现在可以把对象**沿 Z 拖离床面**了);拖平面块 = 在该平面内移动;拖动中光标旁出现琥珀色增量浮标(如「ΔX +12.0 mm」);**按住 Ctrl 拖**,浮标数值以 1mm 步进跳变。松手历史 +1、Ctrl+Z 可撤销;拖到一半按 Esc,对象弹回原位且历史不变。
2. **旋转与缩放**:按 E——三个环,拖环旋转,浮标显示角度,Ctrl 吸附 5°;按 R——轴向方块手柄 + 中心方块,拖轴向手柄单轴缩放、拖中心方块整体缩放(浮标显示倍率与百分比,Ctrl 以 5% 步进);反向拖过头**不会**把对象拖成负尺寸或镜像。
3. **正交轴禁用(VIEW 边界 5)**:按 5 切正交、按 1 进顶视图——蓝色 Z 箭头置灰、点不中拖不动,红/绿箭头与 XY 平面块照常;XZ/YZ 平面块(侧对视线)同样置灰。按 2 进前视图则轮到 Y 向手柄置灰。切回透视(按 5)全部恢复。
4. **沉底(VIEW-06)**:先用 Z 箭头把一个对象拖到半空(或拖进床下),点工具栏琥珀色「⬇ 沉底」——对象底面精确落回 Z=0,历史 +1、Ctrl+Z 可撤销;框选多个对象再点沉底,所有对象**各自**落床,历史仍只 +1。旋转或缩放过的对象沉底同样精确贴床(按变换后的真实包围盒算)。

四条全过 = T6 关闭。

## T7 验收(场景树)

前置:左栏顶部为双页签,「场景树」默认激活(「资产」为 T11 占位);演示场景四个对象成列,其中「已锁定 · 校准板」出厂即锁定。

1. **选中同源(TREE-03)**:树内单击「立方体 30mm」→ 视口同帧出现琥珀描边;反向在视口点圆柱 → 树内对应行高亮并自动滚动入视。Ctrl 点第三行加选(视口出现两个描边)、再 Ctrl 点一次减选;点首行后 Shift 点末行 = 范围选;列表空白处单击 = 清空选中。左下状态条「历史」计数全程**不增加**(选择不入栈,C1)。
2. **三状态列(TREE-02 / C7)**:点某行 👁 → 视口该对象消失、行名变暗,再点恢复;点 🔒 → 对象灰化半透明、视口点不中框不中、Ctrl+A 跳过,但**树内仍可点选、可 F2 重命名**(锁定 ≠ 不可管理)。显隐与锁定任意组合互不干扰。
3. **成组与继承(TREE-04 + C7)**:选中立方体与圆柱 → 点「成组」→ 生成「组 1」,占原首成员位置、自动选中、历史 +1;双击组行(或按 F)→ 相机聚焦两成员的整体包围盒。隐藏组 → 两成员一起消失,而成员自己的 👁 保持原状(行悬停提示「随组隐藏」);锁定组 → 成员视口点不中、gizmo 不出现。点「解组」→ 成员回填原位、历史 +1,Ctrl+Z 后组名、成员顺序完整还原(TREE 边界 2)。
4. **拖拽排序与入组(TREE-04 + 边界 3)**:按住行拖 ≥4px,行间出现琥珀插入线(行高上/下 30% 分界)、光标旁浮标示「移动 N 项」;拖到组名**中段** = 组行亮框,松手入组,历史 +1、Ctrl+Z 回原位;多选拖拽 = 仍只 +1。把对象拖向**锁定的组**中段 → 浮标变红「锁定的组不接受拖入」+ 禁止光标,松手无事发生;把组拖向它自己的子行 → 同样拒绝;拖拽中按 Esc = 取消。拖到原位松手不产生历史记录。
5. **重命名与命名规则(TREE-05)**:点行尾 ✎ 或选中单行按 F2 → 行内输入框,Enter 提交(历史 +1)、Esc 还原、清空提交 = 还原;改成与其他行同名 = **允许**。此后每次「成组」依次得「组 2 / 组 3」;同一资产再次落场自动得「名称 2」(导入去后缀、AI 取 prompt 前 12 字的规则已入 `src/kernel/naming.ts` 单测,T10/T12 联通后界面可见)。
6. **删除与深度软上限(边界 1 + TREE-01)**:选中组按 Del(或点「删除」)= 组及内容一步删除,Ctrl+Z 后全部还原且呈选中;对空组做同样操作(先把成员拖出)可见空组显示「(空)」且不自动消失(边界 6)。连续把组套进组约五层后再拖入对象 → 底部浮出「已超过建议层级深度 5(仅提示,不限制)」,操作照常生效。

六条全过 = T7 关闭。

## T8 验收(参数面板)

前置:右栏为参数面板;左下状态条的「历史」计数是验收 C1 的口径。

1. **三态上下文(PANEL-01/02)**:空白处点击清空选中 → 面板显示「场景设置 · 打印床」,切换预设或改自定义尺寸,视口床网格即时变化且历史**不增加**(床设置不入栈);单选立方体 → 标题为对象名,「变换」展开、「材质」「对象信息」折叠(点标题可展开);框选多个 → 标题「已选 N 项 · M 个对象」。选中一个**组** → 面板即对其成员的多选编辑(与 gizmo 枢轴同口径)。
2. **数值合法性与步进(PANEL-05/06)**:单选对象,旋转 Z 输入 `450` 回车 → 显示 `90.00`,Ctrl+Z **一步**回到编辑前(归一不产生额外记录);位置框输入 `abc` 失焦 → 还原原值、历史不动;缩放输入 `-50` → 变为 `0.10`(0.1% 下限,禁负);聚焦位置 X 按 ↑/↓(±1)、Shift+↑(±10)、Alt+↑(±0.1),连续快按只产生 1 条历史(800ms 合并),停顿超 1 秒再按产生新条目。
3. **多选语义(PANEL-03,验收样例 1)**:选中 3 个 X 不同的对象 → 位置 X 显示**包围盒中心**;输入「中心 +20」的值 → 三个对象整体平移、两两相对距离不变、历史 +1;旋转 Z 各不相同时显示「多值」占位,输入 `45` → 全体绝对统一为 45(各绕自身枢轴,编队旋转记 M2 债)。
4. **%↔mm 双显示与统一锁(PANEL-04 + 边界 3)**:单选立方体(本体 30mm),缩放 X 改 `200` → 尺寸 X 变 `60.00` 且锁开启时 Y/Z 同步等比;反向把尺寸 Z 改 `15` → 缩放联动为 50%。点 🔗 关闭锁 → 只动单轴。多选时「尺寸」行变为只读包围盒(**多选尺寸只读**);多选缩放混合值 + 锁开启,输入 `150` → 全部对象三轴统一为 150%(绝对目标)。
5. **材质与实例覆盖(PANEL-07 / C2)**:展开「材质」,单选改颜色 → 视口即时变色;拖粗糙度滑杆一整段 → 松手历史只 +1(一次拖动 = 一步);同资产的另一个实例颜色不受影响(实例级覆盖);Ctrl+Z 逐步还原。
6. **锁定跳过与编辑期保护(边界 1/2)**:多选含 1 个锁定对象 → 面板底部**常驻**「编辑将跳过 1 个锁定对象」,改颜色后仅未锁定的变色、历史 1 条;把光标停在位置输入框内不失焦,另一只手在**树内**改变选中集,再回车 → 本次编辑被静默丢弃、无报错、历史不动。
7. **拖拽联动(性能注)**:视口里拖 gizmo 移动对象 → 面板位置数值跟随刷新(节流,不逐帧),松手后与 HUD 增量一致。

七条全过 = T8 关闭。

## T9 验收(历史面板)

前置:右侧栏切换到「历史」Tab，纵向时间线按上旧下新排列，头部「N/M」即当前位置指示。

1. **列表与命名规范(HIST-04/07)**:空场景加载后面板示「暂无历史」;随手做几步——拖动一个对象、重命名、锁定、成组——每步在时间线底部新增一枚条目,呈现为**图标 + 操作名 + 目标名**(如 📁 成组 · 组 1 / 🔒 锁定),同一语义图标与场景树一致(👁/🔒/🔓)。
2. **当前位置与前后状态**:Ctrl+Z 两次 → 头部计数从 N/N 变为 N-2/N,被撤销的两条**不消失**,变为下方灰显(redo 侧);当前条目带绿色描边下划线;Ctrl+Shift+Z(或 Ctrl+Y、或 ↷ 按钮)逐步找回。
3. **点击跳转(HIST-04)**:做 5 步操作后点击第 2 枚条目 → 一次性批量撤销到「应用完第 2 步」的状态,视口与场景树同步回退,选中态随每步恢复(HIST-06);点「◦ 初始」→ 全部撤销;再点最后一枚 → 全部重做。跳转前后再做任意新操作 → 右侧灰显条目被静默截断(HIST-01)。
4. **目标名快照**:删除一个对象(或多选删除)→ 该条目显示被删对象的名字(如「支架、支架 2」),撤销前节点已不在文档,名字来自入栈时刻快照;3 个以上折叠为「首名 等 N 项」,悬停 tooltip 看全量。
5. **hover 高亮(HIST-08,P1)**:鼠标悬停任意历史条目 → 视口中受影响对象套**蓝色外壳**(与选中琥珀色区分,选中态下双壳并存可辨);悬停「成组」条目 → 组内全部成员高亮;移开即灭。
6. **入栈口径复核(C1)**:纯点选、转相机、切视角预设 → 时间轴不增;拖 gizmo 一整段 → 只增 1 枚「✥」条目;面板数值快速连按步进 → 只 1 枚「📐」条目(800ms 合并),条目文案随最新值刷新。

注:**预览态冻结呈现**(边界 1:面板灰态可见 + 「预览态 · 历史冻结」角标、条目禁点)已实现并有 SSR 测试覆盖,但 Tab 预览模式的开关在 T18 才接线,当前界面上无法手动触发——T18 验收时一并点测。栈满「更早的记录已合并」占位同理由测试覆盖(手测需 50+ 步操作,不强求)。

六条全过 = T9 关闭。

## T10 验收(解析 Worker · 真实文件导入)

前置:准备测试文件最快的路径是 Blender——① 常规件:默认立方体缩放到 0.05(即 100mm 量级)导出 STL;② 触发单位询问的小件:缩放到 0.002 再导出 STL(数值 2 < 10);③ GLB:缩放到 0.06 导出 glTF Binary(**导出设置里 compression 保持关闭**);④ OBJ:任意件导出 OBJ 并勾选 Materials(产生 mtllib 引用但不带 .mtl 上传)。入口有两个:直接**拖文件进视口**,或工具栏最左的琥珀色「导入」按钮(T10 两入口同语义;T11 资产面板就位后按钮改绑「仅入库」)。

1. **落场链路(IMP-02/04)**:拖入常规 STL → 视口右上出现解析条目(名称 + 阶段 + 进度条),完成后对象**落在床中心、底面贴床、自动选中**,条目补缩略图后约 6 秒自动离场;历史 +1 枚「📥 导入」,Ctrl+Z 实例消失、Ctrl+Shift+Z 找回。
2. **单位三分支(IMP-05)**:
   - 常规 STL(最大边 10–400)→ **无询问**,toast「已按毫米导入」带**「重选单位」**按钮;点它 → 历史新增一枚「🗑️ 删除资产」,弹出单位对话框,改选 cm 确认 → 对象放大 10 倍重新落床(历史再 +1 导入,三步全部线性可撤销)。
   - 小件 STL(数值 2)→ 弹**单位确认对话框**,四选项各带换算后的毫米尺寸、「厘米」标注推荐;床上出现**琥珀色半透明幽灵体**,切换选项实时变大变小;「取消导入」→ 幽灵消失、条目示已取消、**历史与场景零变化**;重拖一次选「米」确认 → 2000mm 巨型件落床(超床只提示不拦截,C4)。
   - GLB → **全程无询问**直接落床,尺寸 = 米数值 × 1000(规范直换;Blender 默认 2m 立方会正当地变成 2000mm,建议按前置说明缩放)。
3. **并发上限与取消(IMP-04/边界 3)**:一次框选 4–5 个文件拖入 → 状态区同时最多 **3 条进行中**,其余排队;对进行中条目点 ✕ → 变「已取消」,排队的下一件立刻顶上。
4. **失败分类(IMP-08/IMP-01)**:拖 .fbx → 失败条目「暂不支持 FBX,请从建模软件导出为 GLB 或 STL」,只有「移除」;把任意 .txt 改名为 .glb 拖入 →「无法解析为有效的 GLB 文件」,带「重试」与「移除」,条目**常驻不自动消失**。
5. **OBJ 降级(IMP-07)**:拖入带 mtllib 引用的 OBJ(不带 .mtl)→ 正常以默认材质落床,toast 提示「OBJ 缺 MTL,已用默认材质」。
6. **主线程零阻塞(IMP-04)**:导入较大文件(可在 Blender 里给球体加两级细分再导 STL)期间连续转动相机、框选 → 无可感知卡顿,进度条持续走动。

注:水密/退化预检结果已随资产元数据写入(焊接 → 邻接分析,单元测试覆盖闭合/开口/退化三类夹具),但检查结果的界面呈现属 T14;资产面板、IndexedDB 持久化与「仅入库」语义属 T11——当前刷新页面导入件即失,属预期。

六条全过 = T10 关闭。

## T11 验收(资产面板与持久化)

2. X / Y / Z 三个中线候选由本地确定性规则生成，不调用大模型；X 中切应得到两件 150 × 80 × 80mm，均显示“可放入”。
3. Y/Z 中切仍保留 300mm 超床轴，两件都显示“仍超床”，不得把无效候选包装成可打印方案。
4. 视口以青/紫区分 A/B，琥珀平面表示当前候选；这里使用 GPU 裁剪显示，不生成截面封口或派生资产。
5. 从 AI 拆件结果点击“比较本地切面候选”会回到打印检查并打开同一预览；Agent 仍无写模型权限。
6. 候选切换、相机和关闭预览全程历史保持 `0/0`；编辑场景或更改打印床后旧预览失效。

详细记录见 [`docs/releases/M1.7.4.md`](docs/releases/M1.7.4.md)。

## 日常协作循环
新会话开工口令:仓库地址 + 任务号(如 T5)+ 遗留问题 → 协作方拉代码开发 → 交付更新 zip → 你按步骤 B 上传 → 自动部署 → 你按 PRD 验收标准点测。
