# 3D STD · Web 工作台

AI 生成 → 编辑摆盘 → 打印检查 → 导出 STL 的网页版 3D 打印创作工作台。
规格见 `docs/`(PRD v0.94 · 技术方案 v1.1 · Backlog v1.0)。当前进度:T1–T7(脚手架 · 编辑内核 · 世界/相机/选择 · Gizmo 与沉底 · 场景树;80 测试)。

## 首次上线(全程网页操作,无需命令行)

**A. 仓库准备**
1. 打开 github.com/bigcatdamao/3D-STD → Settings → General,拉到底部 Danger Zone → Change visibility → **Public**。
   (私有库协作方读不到;若坚持私有,改为每次把代码 zip 传进对话。)
2. 若仓库为空:点 Add file → Create new file,随便建一个 `init.txt` 提交,激活默认分支。

**B. 上传代码**
1. 解压本 zip。
2. 仓库页 Add file → **Upload files**,把解压出的**全部内容**(含 docs、src、worker 等文件夹)一次拖入,Commit。
   备选:打开 Codespace,把 zip 拖进左侧文件树,终端跑 `unzip *.zip && git add -A && git commit -m t1 && git push`。

**C. 绑定 Cloudflare(一次性)**
1. 注册/登录 dash.cloudflare.com(免费档即可)。
2. 左栏 Workers & Pages → Create → **Workers** → Import a repository → 授权 GitHub → 选 `3D-STD`。
3. 构建设置:Build command 填 `npm run build`,Deploy command 保持默认(`npx wrangler deploy`),保存。
4. 等云端构建完成,得到 `https://3d-std.<你的子域>.workers.dev`。

**D. T1 验收(打开上面的 URL)**
- 看到五区布局壳;
- 视口里绿色立方体「立」在打印床网格上(Z-up 正确);
- 顶栏右侧「服务层: 在线」(Worker 路由通)。
三条全过 = T1 完成,此后每次推送代码自动重新部署。

## T5 验收(世界 · 相机 · 选择)

打开线上地址,对照三条:

1. **世界(VIEW-01)**:深色打印床居中,10mm 细网格 + 40mm 主网格,前缘/右缘有「X/Y 256 mm」标注,床边界有半透明成型体积线框;右上角下拉可切 256/180/350/自定义,床与网格随之重算。
2. **鼠标与相机(VIEW-02/03)**:右键拖 = 旋转,中键拖 = 平移,滚轮 = 缩放,左键点对象 = 选中(琥珀描边);键 1/2/3/0 = 顶/前/侧/轴测,F = 聚焦选中,Home = 复位,5 = 透视/正交切换(切换前后画面大小不跳);左下角状态条的「历史」计数在任何相机操作后**不增加**。
3. **选择与死区(VIEW-02/04)**:按住对象拖 ≥3px = 移动(松手后历史 +1,Ctrl+Z 可撤销;拖动中按 Esc = 归位且历史不变);按下几乎不动就松开 = 仅选中、历史不变;空白处拖出框,擦到即选;灰色半透明的「已锁定 · 校准板」点不中、框不中,Ctrl+A 也不选它。

三条全过 = T5 关闭。

## T6 验收(Gizmo 与沉底)

前置:点选任一对象(琥珀描边出现后,视口中出现三色手柄;X 红 / Y 绿 / Z 蓝)。

1. **移动模式(VIEW-05)**:按 W(或工具栏「移动」)——三色箭头 + 三个半透明平面块。沿箭头拖 = 只沿该轴移动(现在可以把对象**沿 Z 拖离床面**了);拖平面块 = 在该平面内移动;拖动中光标旁出现琥珀色增量浮标(如「ΔX +12.0 mm」);**按住 Ctrl 拖**,浮标数值以 1mm 步进跳变。松手历史 +1、Ctrl+Z 可撤销;拖到一半按 Esc,对象弹回原位且历史不变。
2. **旋转与缩放**:按 E——三个环,拖环旋转,浮标显示角度,Ctrl 吸附 5°;按 R——轴向方块手柄 + 中心方块,拖轴向手柄单轴缩放、拖中心方块整体缩放(浮标显示倍率与百分比,Ctrl 以 5% 步进);反向拖过头**不会**把对象拖成负尺寸或镜像。
3. **正交轴禁用(VIEW 边界 5)**:按 5 切正交、按 1 进顶视图——蓝色 Z 箭头置灰、点不中拖不动,红/绿箭头与 XY 平面块照常;XZ/YZ 平面块(侧对视线)同样置灰。按 2 进前视图则轮到 Y 向手柄置灰。切回透视(按 5)全部恢复。
4. **沉底(VIEW-06)**:先用 Z 箭头把一个对象拖到半空(或拖进床下),点工具栏琥珀色「⬇ 沉底」——对象底面精确落回 Z=0,历史 +1、Ctrl+Z 可撤销;框选多个对象再点沉底,所有对象**各自**落床,历史仍只 +1。旋转或缩放过的对象沉底同样精确贴床(按变换后的真实包围盒算)。

四条全过 = T6 关闭。

## T7 验收(场景树)

前置:左栏顶部为双页签,「场景树」默认激活(「资产」为 T11 占位);演示场景四个对象成列,其中「已锁定 · 校准板」出厂即锁定。

1. **选中同源(TREE-03)**:树内单击「立方体 30mm」→ 视口同帧出现琥珀描边;反向在视口点圆柱 → 树内对应行高亮并自动滚动入视。Ctrl 点第三行加选(视口出现两个描边)、再 Ctrl 点一次减选;点首行后 Shift 点末行 = 范围选;列表空白处单击 = 清空选中。左下状态条「历史」计数全程**不增加**(选择不入栈,C1)。
2. **三状态列(TREE-02 / C7)**:点某行 👁 → 视口该对象消失、行名变暗,再点恢复;点 🔒 → 对象灰化半透明、视口点不中框不中、Ctrl+A 跳过,但**树内仍可点选、可 F2 重命名**(锁定 ≠ 不可管理)。显隐与锁定任意组合互不干扰。
3. **成组与继承(TREE-04 + C7)**:选中立方体与圆柱 → 点「成组」→ 生成「组 1」,占原首成员位置、自动选中、历史 +1;双击组行(或按 F)→ 相机聚焦两成员的整体包围盒。隐藏组 → 两成员一起消失,而成员自己的 👁 保持原状(行悬停提示「随组隐藏」);锁定组 → 成员视口点不中、gizmo 不出现。点「解组」→ 成员回填原位、历史 +1,Ctrl+Z 后组名、成员顺序完整还原(TREE 边界 2)。
4. **拖拽排序与入组(TREE-04 + 边界 3)**:按住行拖 ≥4px,行间出现琥珀插入线(行高上/下 30% 分界)、光标旁浮标示「移动 N 项」;拖到组名**中段** = 组行亮框,松手入组,历史 +1、Ctrl+Z 回原位;多选拖拽 = 仍只 +1。把对象拖向**锁定的组**中段 → 浮标变红「锁定的组不接受拖入」+ 禁止光标,松手无事发生;把组拖向它自己的子行 → 同样拒绝;拖拽中按 Esc = 取消。拖到原位松手不产生历史记录。
5. **重命名与命名规则(TREE-05)**:点行尾 ✎ 或选中单行按 F2 → 行内输入框,Enter 提交(历史 +1)、Esc 还原、清空提交 = 还原;改成与其他行同名 = **允许**。此后每次「成组」依次得「组 2 / 组 3」;同一资产再次落场自动得「名称 2」(导入去后缀、AI 取 prompt 前 12 字的规则已入 `src/kernel/naming.ts` 单测,T10/T12 联通后界面可见)。
6. **删除与深度软上限(边界 1 + TREE-01)**:选中组按 Del(或点「删除」)= 组及内容一步删除,Ctrl+Z 后全部还原且呈选中;对空组做同样操作(先把成员拖出)可见空组显示「(空)」且不自动消失(边界 6)。连续把组套进组约五层后再拖入对象 → 底部浮出「已超过建议层级深度 5(仅提示,不限制)」,操作照常生效。

六条全过 = T7 关闭。

## 日常协作循环
新会话开工口令:仓库地址 + 任务号(如 T5)+ 遗留问题 → 协作方拉代码开发 → 交付更新 zip → 你按步骤 B 上传 → 自动部署 → 你按 PRD 验收标准点测。

## 本地/云端开发(可选)
Codespace 终端:`npm i && npm run dev`,按提示打开转发端口即可看到实时画面。
