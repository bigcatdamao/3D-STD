## 1. 背景与定位

**一句话定位**:AI 创作到打印就绪的一站式上游工作台——从 AI 生成 / 文件导入,到编辑摆盘,到打印前检查与导出,在一个网页里完整走通;终点是导出打印就绪文件,无缝交接切片生态(Bambu Studio)。

**范围红线**(v0.94 定位修订):切片引擎与打印机控制**不做**——前者是切片软件厂商多年积累的核心资产,后者依赖私有协议,M1 硬啃这两段的结局是五段全都半生不熟。本产品的站位是切片软件的**上游**:把 AI 创作流量以打印就绪的形态导入切片生态,而非替代它;本地导入与 AI 落入共用同一条解析管线(T10),导入是 AI 模型进场的必经之路开出的第二入口,不是对切片软件的功能复刻。M1 终点 = 导出的 STL 在 Bambu Studio 打开尺寸一致(CP3 验收标准);切片预览与局域网发送打印列 M2/M3 路线图。

**为什么做**:AI 3D 生成的产出与真实打印流程之间存在断层。对 5 个主流 AI 3D 生成平台、30 个样本的实测评估表明,生成模型无法进入打印流程的主因高度集中:非水密网格、退化几何、尺寸失真、悬空与超打印体积。现有工具要么是纯生成平台(不管打印),要么是纯切片软件(不管生成)。本产品把「生成 → 编辑 → 打印就绪」做进同一条工作流,并将上述失败模式产品化为内建检查器。

**目标用户**:消费级 3D 打印用户 / 创客。特征:熟悉切片软件(Bambu Studio、PrusaSlicer)的交互习惯,不必熟悉专业建模软件;关心「打出来是什么样、能不能打」,而非渲染效果。

**设计信条**(贯穿全文,冲突时以此裁决):
1. 交互习惯对齐切片软件,不对齐建模软件——用户的肌肉记忆来自 Bambu Studio,不来自 Blender。
2. 约束靠反馈,不靠拦截——警告、确认、可视化提示,不替用户做「不许」的决定。
3. 昂贵的结果不可销毁——消耗了配额与等待时间的 AI 生成物,在任何操作路径下都不应被静默毁掉。
4. 可预测性优先于聪明——行为宁可朴素一致,不做依赖上下文猜测的「智能」默认。

## 2. 目标与成功标准

**产品目标**:访客可在线完成一次完整创作闭环——生成或导入模型 → 编辑摆盘 → 通过打印检查 → 导出可被 Bambu Studio 正常打开的 STL。

**成功标准**:
- S1 首次访客经示例项目在 3 分钟内触达全部核心模块(视口、场景树、参数面板、检查、导出),零配额消耗;
- S2 AI 生成任务从提交到结果落入场景,全链路状态可见,任一失败分支均有明确的分类提示与出路;
- S3 导出的 STL 在 Bambu Studio 中打开后尺寸与站内标注一致(mm),无坐标系翻转;
- S4 公开上线状态下,单访客生成配额受控,API 成本上限可预测。

### 3.2 非目标(Non-goals,含理由)

| 不做 | 理由 |
|---|---|
| 真实切片引擎 | 切片是切片软件的本职;本产品的边界是「切片前就绪」 |
| 网格自动修复(补洞) | 浏览器端修复结果不可控;宁可引导至切片软件的成熟修复能力 |
| 建模 / 雕刻能力 | 创作来源定位为 AI 生成与导入,不与建模软件竞争 |
| 账号体系 | 以本地持久化 + 访客配额替代;上线成本与隐私负担不成比例 |
| 协作 / CRDT 合并 | 单人创作工具;多标签冲突以只读模式解决 |
| 历史树(分支撤销) | 认知成本高于收益;线性栈为行业共识 |
| 自动 LOD / 网格简化 | 以导入面数警告替代;简化质量不可控 |
| 回收站 / 软删除 | 本地存储预算宝贵;以删除确认替代 |

## 5. 全局规则(宪法)

以下规则为跨模块约束,各模块需求默认引用,不再重复展开。

**C1 操作原子性**。一切用户操作归入三类:
- 立即入栈:结构性操作(导入落场、删除、成组/解组、重命名、显隐、锁定),一动作一记录;多选批量操作合并为一步。
- 合并入栈:连续型操作,以交互会话为一步——gizmo 拖拽以 pointer-down 至 pointer-up 为界;滑杆同理;数值输入以失焦或回车提交为界;同一参数 800ms 内连续键入合并。
- 不入栈:选择、相机、视图切换、导出等不改变文档状态或只读的操作。

**C2 资产 / 实例分离**。资产 = 源数据(几何、材质、元数据、生成参数),存于全局资产库;实例 = 场景中对资产的引用 + 变换 + 材质覆盖。删除实例不销毁资产;资产级属性(水密性、退化几何)检查一次缓存于资产,实例级属性(床外、悬空)随变换重算。

**C3 坐标系与单位**。世界坐标 Z-up,单位 mm,打印床为世界原点与地面。glTF(Y-up、米)在导入时一次性烘焙转换;STL 导出零转换直写。

**C4 只提示,不拦截**。床外、非水密、悬空等问题以警告呈现(视口 + 场景树 + 检查面板三处同步),不禁止操作;导出遇错误级问题弹确认而非禁用。

**C5 持久化三层**。运行内存:历史栈、选中态,刷新即清;IndexedDB:资产库、项目场景,跨会话;云端:AI 生成结果转存,本地缓存丢失可回源。IndexedDB 不可用时全站降级会话模式并常驻提示。

**C6 输入通道分工**。参数面板 = 绝对值语义;gizmo = 相对增量语义。多选混合值时面板输入即绝对统一,混合态禁 scrub。

**C7 对象三状态正交**。选中、显隐、锁定独立存储、任意组合。隐藏 = 不渲染、不参与检查、不导出;锁定 = 视口不可选不可变换,场景树内可管理;组状态对成员为继承式叠加,成员自身标记保留。

## 6. 功能需求

### 6.1 项目与顶栏(PROJ)

| 编号 | 需求 | 优先级 |
|---|---|---|
| PROJ-01 | 欢迎页:新建空项目、最近项目列表(缩略图+修改时间)、示例项目入口 | P0 |
| PROJ-02 | 示例项目:预置含 AI 资产、床外警告、非水密对象的场景;每次打开自模板复制;全程零配额 | P0 |
| PROJ-03 | 自动保存:编辑停止 2s 防抖写入 IndexedDB;保留 Ctrl+S 立即落盘并反馈 | P0 |
| PROJ-04 | 保存状态常驻指示:已保存 / 保存中 / 会话模式三态 | P0 |
| PROJ-05 | 项目重命名(联动导出默认文件名)、删除(确认+不可恢复声明)、切换(先 flush 后切) | P0 |
| PROJ-06 | 顶栏布局:左「项目态」/ 中「过程」(撤销重做、导入、编辑-预览切换)/ 右「出口」(检查、导出主 CTA、注释层开关) | P0 |
| PROJ-07 | 多标签同项目:BroadcastChannel 互探,后开标签只读+横幅+「在此接管」 | P1 |

**边界情况**:
1. 有未落盘更改关闭页面:beforeunload 提示;已保存状态不拦。
2. 项目名清洗:剔除文件系统非法字符,限长 60;允许重名(ID 为唯一标识)。
3. 加载遇资产缺失:实例渲染占位线框 + 场景树标红(见 AST 边界 4)。
4. 撤销/重做按钮态与历史栈实时同步,空栈置灰。
5. 自动保存写入失败:降级会话模式,编辑不中断。

**验收标准样例**:
- 编辑后静置 2s 刷新页面,场景完整恢复且顶栏示「已保存」;编辑后 1s 内刷新,出现 beforeunload 提示。
- 第二标签打开同一项目进入只读并显示横幅;点击「在此接管」后两标签角色互换。
- 示例项目内完成「修复床外 → 检查通过 → 导出 STL」,配额消耗为 0。

### 6.2 文件导入与解析(IMP)

| 编号 | 需求 | 优先级 |
|---|---|---|
| IMP-01 | 支持 GLB/glTF、STL(二进制+ASCII)、OBJ 导入;3MF 为 P1;FBX/STEP 拒绝并给出替代建议文案 | P0 |
| IMP-02 | 三入口两语义:拖入视口 = 入库+建实例(床中心+自动沉底);拖入资产面板 / 文件选择器 = 仅入库 | P0 |
| IMP-03 | 前置校验:扩展名白名单、单文件 ≤100MB、面数 ≤200 万(超面数警告不拒绝) | P0 |
| IMP-04 | Web Worker 异步解析,进度反馈,并发上限 3,主线程零阻塞 | P0 |
| IMP-05 | 单位推断置信度分级:包围盒最大边 L∈[10,400] 静默按 mm+可撤 toast;否则弹单位确认(mm/cm/inch/m,床上实时预览);glTF 按规范米→mm 直换不询问 | P0 |
| IMP-06 | 坐标系烘焙:glTF Y-up → 世界 Z-up,导入时一次完成(C3) | P0 |
| IMP-07 | 后处理:网格统计(面数/顶点/包围盒)、水密性预检(前置按距离 ε 焊接顶点后再做邻接分析——STL 逐三角形独立存储顶点,不焊接将全量误报;写入资产元数据,不弹窗)、OBJ 缺 MTL 降级默认材质+标记、离屏缩略图 | P0 |
| IMP-08 | 解析中占位条目(进度可见)、失败态(分类文案+可重试/移除),不静默消失 | P0 |

**边界情况**:
1. 重复导入同名文件:允许,命名加序号;内容 hash 去重为 P2。
2. 损坏 / 伪装扩展名:进失败态,文案分类(「无法解析为有效的 STL」)。
3. 解析中删除占位:等同取消,Worker 终止。
4. 零字节 / 空网格:按失败处理,「文件不包含有效几何」。
5. 单位选择写入资产元数据,同资产复用不再询问。

**验收标准样例**:
- 导入 50MB 二进制 STL 期间连续操作相机无可感知卡顿,占位进度持续更新。
- 按米建模的 GLB(包围盒 0.12m)拖入视口,床上实例实际 120mm,全程无单位询问。
- 缺 MTL 的 OBJ 以默认材质渲染,资产条目示「材质缺失」,检查与导出不受影响。

### 6.3 资产面板(AST)

| 编号 | 需求 | 优先级 |
|---|---|---|
| AST-01 | 左栏与场景树双页签;网格视图 + 缩略图 + 来源角标(导入/AI) | P0 |
| AST-02 | 资产条目结构:ID、名称、来源、几何与材质本体、元数据(面数/包围盒/单位/水密性/时间)、缩略图、状态(解析中/就绪/失败/失效);AI 资产附完整生成参数(prompt、参考图、引擎、配置) | P0 |
| AST-03 | 拖入视口建实例、重命名、删除(实例级联确认,列出受影响项目与实例数) | P0 |
| AST-04 | 持久化遵循 C5;本地上限 500MB,80% 提示,超限拒写并引导清理(禁止自动 LRU 淘汰) | P0 |
| AST-05 | 云端转存:仅 AI 生成资产;本地缓存丢失自动回源恢复 | P0 |
| AST-06 | 搜索、排序(时间/名称/面数)、详情弹层(元数据+生成参数;失效条目同样可查) | P1 |
| AST-07 | AI 资产「生成变体」:原参数回填生成输入 | P1 |

**边界情况**:
1. IndexedDB 不可用:纯内存会话模式 + 常驻提示「本次会话的资产不会被保存」。
2. 本地丢、云端在:条目标「需恢复」,使用时无感回源。
3. 双失效:转「失效」态,保留元数据与生成参数,提供「按原参数重新生成」(耗配额,需确认)。元数据永久保留。
4. 项目引用缺失资产:占位线框(警告色)+ 树标红,不静默丢对象。
5. 失败态条目删除:无确认直接移除。
6. 资产库全局共享、项目仅引用;删除级联确认需列出跨项目影响。

**验收标准样例**:
- AI 资产落库后强制刷新,条目完整(缩略图/元数据/生成参数),拖入视口不耗配额。
- 清空本地库后点击该 AI 资产,自动云端恢复并可建实例,无报错。
- 隐私模式下完成导入-编辑-导出全流程,会话模式提示常驻,无功能阻断。

### 6.4 场景树(TREE)

| 编号 | 需求 | 优先级 |
|---|---|---|
| TREE-01 | 层级:扁平列表 + 可嵌套组,深度软上限 5(超出提示不禁止) | P0 |
| TREE-02 | 三状态图标列(显隐/锁定),行为遵循 C7 | P0 |
| TREE-03 | 多选:Ctrl 加减选、Shift 范围选;与视口共享同一选中集(全站单一事实源) | P0 |
| TREE-04 | 成组/解组、重命名、删除、拖拽排序与入组;双击节点视口聚焦 | P0 |
| TREE-05 | 命名规则表:导入=文件名去后缀;AI=prompt 前 12 字符;实例继承资产名;重复加序号;组「组 N」;允许重名 | P0 |
| TREE-06 | 检查联动:床外/非水密对象黄标 | P1 |
| TREE-07 | 搜索过滤、右键菜单、solo 孤立显示 | P1 |

**边界情况**:
1. 删除组:仅提供「删除组及内容」(一步入栈)与「解组」两个明确操作。
2. 解组撤销对称:恢复组名、成员顺序与状态标记。
3. 锁定组不接受拖入,悬停示禁止光标。
4. AI 落入一律根层级 + 自动选中 + 视口聚焦(可预测性,信条 4)。
5. 多选变换 pivot = 共同包围盒中心。
6. 空组保留显示空态,不自动删除(保护撤销对称)。
7. 对象 ID 在删除-恢复循环中保持稳定(历史栈引用完整性的前提)。

**验收标准样例**:
- 隐藏实例后导出,文件不含其几何。
- 锁定对象无法被视口框选;在树中可选中并重命名。
- 删除含 3 实例的资产并确认:树移除 3 实例、历史仅 1 条;撤销后资产、实例及组内位置全部还原。

4. 拖动中 Esc:取消归位,不入栈。
5. 正交视图沿视线轴的 gizmo 把手置灰禁用。
6. 面板折叠 / resize:视口重算,相机与选中态不变。

**验收标准样例**:
- 对象上按下位移 2px 后松开(低于 3px 死区阈值):未移动、被选中、历史无新记录;位移 10px(高于阈值):移动且新增 1 条。
- 对 Z 最低点 -3mm 的对象沉底:底面归零、床外警告消除、可撤销。
- 预览态下点选无效、Ctrl+Z 无效;切回编辑态选中集与历史栈完整。

**性能底线**:总面数 100 万时相机操作 ≥30fps(桌面中端核显,待校准);gizmo 拖拽期间面板数值节流刷新。

| PANEL-03 | 多选语义:位置=包围盒中心,编辑=整体平移保持相对位置;旋转/缩放=围绕共同中心各自应用;混合值示「多值」占位,输入即绝对统一(C6) | P0 |

| PANEL-05 | 合法性:缩放 clamp ≥0.1%、禁负(镜像为显式操作,P1);旋转归一 (-180,180],欧拉序固定 XYZ;位置不 clamp(床外走警告);非法输入失焦还原;显示 2 位小数、存储全精度、舍入不回写 | P0 |

**边界情况**:
1. 多选含锁定:可编辑,提交跳过锁定成员,底部常驻提示「编辑将跳过 N 个锁定对象」。

3. 统一锁 + 混合值:输入的百分比作为绝对目标统一应用(与 PANEL-03 一致)。
4. 混合态禁 scrub(增量无起点定义)。

**验收标准样例**:
- 多选 3 个 X 不同的对象:位置 X 显示包围盒中心;+20 后两两相对距离不变。
- 旋转输入 450:失焦显示 90;撤销一步回到编辑前(归一不产生额外栈记录)。
- 多选含 1 锁定对象改颜色:仅 2 个实例变色、历史 1 条、撤销同步还原。

### 6.7 历史栈(HIST)

| 编号 | 需求 | 优先级 |
|---|---|---|
| HIST-01 | 线性栈,撤销后新操作静默截断 redo 分支;深度上限 50(待校准),满丢最老 | P0 |
| HIST-02 | Command 模式记录 {操作类型, 目标ID, before, after},不做场景快照;删除类暂存完整对象数据并设内存上限 | P0 |
| HIST-03 | 入栈规则全量遵循 C1 | P0 |
| HIST-04 | 可视化历史面板:图标+操作名+目标名列表、当前位置指示、点击跳转(批量撤销/重做)、Ctrl+Z / Ctrl+Shift+Z | P0 |
| HIST-05 | AI 特殊规则:生成任务不入栈,仅「落入场景」入栈;撤销=移除实例,资产保留(C2) | P0 |
| HIST-06 | 撤销恢复操作前选中态(如撤销删除后对象呈选中) | P0 |
| HIST-07 | 条目命名规范表(操作类型 → 显示文案) | P0 |
| HIST-08 | hover 条目视口高亮受影响对象 | P1 |

**边界情况**:
1. 预览态:历史栈冻结、面板灰态可见,回编辑态解冻。
2. 生成中允许其他编辑:异步任务与历史栈互不阻塞。
3. 撤销「导入」:实例移除,资产保留(与 HIST-05 同理)。
4. 撤销「解组」:组结构、成员顺序完整还原;依赖对象 ID 稳定(TREE 边界 7)。
5. 栈满不提示,面板顶部示「更早的记录已合并」占位。
6. 任何撤销/重做后,打印检查结果标过期,不尝试同步回滚。

**验收标准样例**:
- 连续拖拽 gizmo 3 秒松手:历史新增且仅新增 1 条。
- 撤销「删除(多选 3 对象)」:全部恢复且呈选中,树内层级与删除前一致。
- 撤销「AI 生成落入」:资产面板该模型仍在;再次拖入不触发生成、不耗配额。

### 6.8 AI 生成链路(AI)

| 编号 | 需求 | 优先级 |
|---|---|---|
| AI-01 | 入口嵌于工作流(空场景中央创作面板，已有模型时由顶栏再次打开),支持文字 / 单图 / 多图三模式;单图提供大面积点击/拖拽区,多图当前最多三张(正面必填 + 左侧/右侧),前端即时校验(空输入、图片格式/大小/数量),失败不消耗配额 | P0 |
| AI-02 | 服务层代理:密钥隔离、任务提交、配额扣减 | P0 |
| AI-03 | 状态机:输入 → 提交 → 排队(位置反馈)→ 生成中(轮询进度)→ 预览确认(接受/调整/丢弃)→ 落入场景树 | P0 |
| AI-04 | 轮询:排队 5s / 生成 2s 分频 | P0 |
| AI-05 | 失败三分类三出路:超时→重试;审核拒绝→修改输入;服务端引擎额度/服务异常→用户侧示「服务暂时不可用,请稍后再试」并触发运营告警(不暴露账务细节)。三类均返还配额,禁止合并为单一失败弹窗 | P0 |
| AI-06 | 排队与生成阶段均可取消 | P0 |
| AI-07 | 配额规则,成本归因原则:服务侧问题(超时、审核、引擎异常)不让用户买单,用户自身选择(丢弃、重生成)才计费。具体:提交扣、取消还、失败还、成功后丢弃不还;访客配额不足属提交前拦截(不进入扣减),出路「明日再来 / 自带 API key」 | P0 |
| AI-08 | 「调整」保留完整上下文(prompt/图片元数据/参数)回填输入,仅改差异;本地图片二进制不持久化,刷新后需重新选择 | P0 |
| AI-09 | 落入:实例入根层级+选中+聚焦+自动沉底+自动跑首次检查;结果异步转存云端(AST-05) | P0 |
| AI-10 | 引擎抽象为统一生成协议,前端不感知具体引擎;M1 接 Tripo,M2 双引擎可切换 | P0/P1 |
| AI-11 | 防滥用:访客生成配额 3 次/日(§9),示例项目承接零配额体验;「自带 API key」解锁通道;演示码(URL 令牌,提升配额、可撤销)供面试/演示链接使用 | P0 |

**边界情况**:
1. 生成中刷新页面:任务凭本地任务 ID 恢复轮询;不可恢复则按超时处理并返还配额。
2. 转存失败:结果仍可用(平台临时 URL),条目标「未备份」,后台重试。
3. 并发任务上限 1(单访客),排队语义由平台侧队列承接。

**验收标准样例**:
- 断网模拟超时:出现「重试」出路,配额返还,重试成功后正常落入。
- 生成成功后选「调整」:输入区呈原 prompt 与参数,仅修改后重新提交。
- 落入后对象贴床、被选中、相机完成聚焦过渡,树中出现首检结果标记。
- 单图点击区选择 PNG 后出现预览,可替换/删除;GIF 与单张 >10MB 在请求前拦截。
- 多图以“正面 + 左侧”两张即可提交,第三张右侧可选;服务层按 Tripo `[front,left,back,right]` 四槽协议补空 back。

### 6.9 打印检查与导出(CHK)

| 编号 | 需求 | 优先级 |
|---|---|---|
| CHK-01 | 三级检查:错误(非水密、退化几何、确定自交、超出打印体积)/ 警告(内部壳、孤立碎片、深度扫描未完整、悬空、微小件;悬垂为 P1)/ 信息(面数、逐对象尺寸、连通壳、清单) | P0 |
| CHK-02 | 触发:手动按钮 + 导出前自动;Web Worker 执行、带进度、30s 超时按未完成呈现 | P0 |
| CHK-03 | 生命周期:任何编辑后结果标过期(灰显+重新检查),不自动重跑 | P0 |
| CHK-04 | 分层计算:资产级(水密/退化/连通壳/自交)一次缓存;实例级(床外/悬空/微小)随变换重算(C2)。深度自交扫描必须有预算并显式返回完整/部分状态 | P0 |
| CHK-05 | 结果面板分级列表;点击条目 → 视口聚焦 + 问题区域高亮(非水密边描红、悬空示投影距离、自交红蓝三角形对逐组浏览、多个连通壳分色并逐壳聚焦、单壳超床时比较 X/Y/Z 切面并在 10%–90% 调整位置);树黄标联动。切面必须实时显示 A/B 估算尺寸、床适配和可放入区间；证据与切割预览只改临时 UI/相机，不入历史，不得冒充闭合切割结果 | P0 |
| CHK-06 | 确定性修复:悬空→沉底;超床→移回最近合法位;非水密/退化→安全修复预览后生成派生副本。预览支持新增面绿色/删除面红色差异；均入栈可撤销，原资产不改写；复杂拓扑明确拒绝 | P0 |
| CHK-07 | 导出 STL(二进制):范围=全部可见/仅选中;多对象默认合并单网格,可选逐对象 zip;面板注明「STL 不保留颜色与材质;保留对象结构与颜色请用 3MF」 | P0 |
| CHK-08 | 导出遇错误级:确认框列明问题,确认后放行(C4) | P0 |
| CHK-09 | 3MF(对象+颜色结构)、GLB 导出;悬垂角热图(>45°) | P1 |

**边界情况**:
1. 悬空判定(M1 定义):对象 Z 最低点 > 0.5mm(待校准)即报警告;不做对象间堆叠支撑的 mesh-mesh 接触判定——该判断属切片器职责,超出本产品边界。
2. 检查中删对象:相关条目随对象失效移除。:对象 Z 最低点 > 0.5mm(待校准)即报警告;不做对象间堆叠支撑的 mesh-mesh 接触判定——该判断属切片器职责,超出本产品边界。
3. 空场景 / 全隐藏:导出置灰+提示。
4. 仅选中但含隐藏:隐藏优先排除,确认框注明。
5. 超时:允许分对象重试,不假装成功。
6. AI 资产落入即首检(AI-09),生成物质量透明化。

**验收标准样例**:
- 1 个非水密资产 × 6 实例:结果报 6 条错误,几何分析仅执行 1 次(耗时日志验证)。
- 点击悬空条目:聚焦+投影距离显示;修复后贴床、警告消除、历史 1 条可撤销。
- 含错误级问题导出:确认框出现;导出文件在 Bambu Studio 打开尺寸一致(mm)、无翻转。
