# 3D-STD AI 拆件分析 Gold Set 评测计划

| 项 | 内容 |
|---|---|
| 版本 | v0.1 · M1.6.3 |
| 目标样本 | 24 个，覆盖 6 类任务，每类至少 3 个 |
| 当前种子 | 5 个场景定义；内置示例已跑真链，其余待补真实模型 |
| 自动评分实现 | `src/agent/split-analysis-eval.ts` |
| 数据契约 | `docs/contracts/split-analysis-gold.schema.json` |
| 种子清单 | `docs/evals/split-analysis-gold-v0.1.json` |

## 1. 为什么要评测

阶段一不是验证“接口能返回文字”，而是确认用户能否据此做出正确、安全的拆件决策。评测必须同时覆盖结论、证据、限制、方案可用性、行动性、延迟和成本，任何单一演示成功都不能代表阶段一关闭。

## 2. 样本结构

24 个样本按以下六类均衡分配：整体可打印、尺寸超限、网格阻断、支撑敏感、已有分件装配、主要外观面敏感。每个样本保存原始模型路径、打印床与参数、人工基准结论、允许引用的证据 ID、建议件数区间和必须披露的能力限制。

模型文件不进入公开仓库；Gold JSON 只记录相对路径或受控资产 ID。若模型涉及用户数据或商业版权，进入评测集前必须单独确认授权。

## 3. 自动评分（100 分）

| 指标 | 分值 | 规则 |
|---|---:|---|
| 是否拆件结论 | 35 | 命中人工允许结论集合 |
| 必需原因覆盖 | 20 | 尺寸超限、非水密等阻断原因不得漏报 |
| 证据完整性 | 20 | 所有 evidenceRefs 必须来自该样本允许列表；出现伪造引用整项 0 分 |
| 建议件数 | 10 | preferred 落入人工区间 |
| 能力限制诚实度 | 10 | 未运行的薄壁、过悬、切面等必须披露 |
| 方案结构 | 5 | 保持 2–3 套候选方案 |

自动通过要求：总分至少 80、结论命中、伪造证据为 0。

## 4. 人工评分（每项 1–5）

- 方案可用性：是否能真实指导打印与分件，而不是泛泛建议。
- 行动性：用户能否明确知道下一步点哪里、检查什么、为何这么做。
- 表面风险诚实度：是否避免把视觉猜测包装成精确几何结论。
- 装配清晰度：接缝、定位、强度与装配代价是否讲清楚。
- 幻觉标记：任何不存在的对象、问题、切面、检测结论或工具执行都记为 1 次幻觉。

## 5. 阶段一关闭门槛

- 24 个样本全部完成双人复核。
- 拆件结论准确率 ≥ 85%，尺寸超限阻断漏报为 0。
- 证据完整率 ≥ 95%，人工幻觉率 ≤ 5%。
- 自动通过率 ≥ 80%，人工“方案可用性与行动性均 ≥ 4”的比例 ≥ 80%。
- p95 延迟 ≤ 45 秒；记录平均总 token，并在同等质量下优先选择成本更低配置。
- 未达到门槛时继续改提示词、Schema、证据或 UI，不进入 Agent 自动工具循环。

## 6. 执行顺序

1. 补齐 19 个已授权真实模型，冻结模型文件哈希和打印参数。
2. 两名评审独立填写人工基准，分歧由第三次复核裁决。
3. 每个模型使用相同 Provider、模型、推理强度和图片规格运行至少 3 次。
4. 保存结构化输出、Provider、模型、耗时和 token；禁止保存 API Key。
5. 用自动评分器生成汇总，再补人工四项评分与幻觉标记。
6. 对失败样本按“证据不足 / 提示词 / 模型能力 / UI 行动性 / 几何能力缺失”归因。
