## 1. 阶段判断

T1–T16 已打通“AI 生成 / 文件导入 → 编辑摆盘 → 打印检查 → STL 导出”，工程完整度已经不是主要短板。M1.5 的主问题转为：

1. 视口、场景、属性、检查、AI、历史同时争夺注意力，界面层级偏工程工具。
2. 新用户缺少清晰的首次入口与下一步提示。
3. 高面数 AI 资产让本地存储快速逼近 500MB，资产查找和清理效率不足。
4. 顶栏仍有开发占位文案，图标、间距、颜色和控件状态缺少统一系统。
5. 项目生命周期和示例路径尚未实现，无法形成“回来继续做”的产品体验。

T20 产品视频暂停。后期视频以最终稳定 UI 的真实录屏和高质量动效为素材，不作为当前功能设计的替代物。

## 2. M1.5 产品原则

- **视口优先**：3D 内容始终是第一视觉焦点，辅助面板可收纳且记住用户偏好。
- **任务导向**：界面围绕“开始 → 编辑 → 检查 → 导出”组织，不暴露开发任务编号。
- **渐进披露**：属性与检查不同时占满右栏；历史与 AI 可按需收起。
- **问题可行动**：检查结果必须告诉用户对象、影响和下一步，确定性修复直接提供按钮。
- **数据可控**：资产搜索、排序、占用提示和显式删除形成完整的存储管理闭环。
- **宪法不回退**：M1.5 只改产品体验，不破坏 C1–C7、资产/实例分离和检查/导出语义。

## 4. 本批验收清单

1. 三个辅助区域可独立收起/展开，刷新后布局偏好保留。
2. 收起左右栏和底栏后，视口明显扩展，场景、选择、相机和历史不受影响。
3. 点顶栏“打印检查”会展开右栏并显示检查页；检查仍遵循警告不阻断、错误导出需确认。
4. 资产页输入名称或“AI”可过滤；排序可在最近、名称、面数之间切换；不改资产和场景数据。
5. 删除所有实例后出现双入口空态；“导入并放置”直接落床，“使用 AI 生成”展开并聚焦输入区。
6. 原 T1–T16 自动化测试、TypeScript 检查和生产构建继续全绿。

## 5. 暂不进入本批的内容

- R2 云备份（T13b）：仍低于项目生命周期、资产管理和核心体验优先级。
- 切片、自动补洞、打印机控制：继续遵守产品上游工作台边界。
- 视频生成：待 M1.5 UI 和核心路径稳定后单独制作。
