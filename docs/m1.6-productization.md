# 3D Studio · M1.6 AI 拆件助手产品化路线

| 项 | 内容 |
|---|---|
| 版本 | v0.1 |
| 日期 | 2026-07-21 |
| 目标 | 先建立可信、可解释、只读的拆件决策体验，再逐步开放预览和确认后执行 |
| 当前批次 | M1.6.1 只读交互与本地 Mock 原型已完成 |

## 1. 产品定位

M1.6 不是“一键自动拆件”，而是把已有场景、打印空间和网格检查结果组织成用户能够判断的方案。第一阶段只做分析和建议；第二阶段才提供可丢弃的切割预览；第三阶段必须在用户确认后执行，并具备原始副本、历史记录和撤销保障。

## 2. 版本拆分

### M1.6.1 · 只读体验原型（已完成）

- 右侧检查器新增“AI 拆件”Tab，打印检查完成页增加入口。
- 自动汇总对象数、世界尺寸、打印床、检查状态和现有拓扑问题。
- 提供拆件目标、打印方式和四类优先目标。
- 本地 Mock 返回“是否拆件、原因、建议件数、候选方案、优缺点、风险、下一步”的完整产品结构。
- 缺失薄壁、局部过悬、候选切面和多视角证据时如实显示能力限制。
- 场景变化后结果立即过期；分析不修改场景、不写操作历史。

### M1.6.2 · 真实 Responses API 闭环（下一批）

- 新增后端 `/api/agent/split-analysis`，API Key 仅放 Worker Secret。
- 前后端输入输出统一使用严格 JSON Schema，拒绝额外字段。
- 服务端实现超时、refusal、incomplete、离线规则摘要和可观测错误码。
- 增加幂等、缓存、单用户日限额、全局熔断和 usage 记录。
- UI 把“本地 Mock”替换为真实服务状态，并保留明确的数据限制和结果来源。

### M1.6.3 · 多视角与质量评测

- 生成前/左/右/轴测固定构图截图，不包含 UI、Gizmo 或隐私信息。
- 建立 20–30 个代表模型的 Gold Set，评估需要拆件判断、方案可用性、幻觉率、延迟和成本。
- 达到预设门槛后再关闭阶段一；未达标时不进入预览工具开发。

### M1.7 · 只读工具与预览

- 实现 `inspect_model`、`find_cut_candidates`、`preview_plane_cut`、`compare_split_schemes`、`analyze_print_orientation`。
- 预览进入独立临时层，场景变化即失效，不能写入原始模型和历史。

### M1.8 · 确认后执行

- 在 copy-on-write、审批票据、原子历史和撤销全部就绪后，再开放切割、合并、定位销和装配验证。
- 每次写操作必须遵循“预览 → 用户确认 → 执行 → 验证 → 历史入栈”。

## 3. M1.6.1 关闭标准

- 用户能从打印检查自然进入 AI 拆件分析。
- 结果字段完整，并能比较候选方案。
- 不把整件离床误称为局部过悬，不把未检测能力显示为通过。
- 不生成伪精确坐标或伪切割预览。
- 分析前后场景版本、对象数和历史长度不发生变化。
- 自动化测试、类型检查、生产构建和真实浏览器验收通过。

## 4. 下一步优先级

1. Worker 端点和安全边界。
2. 严格输入/输出 Schema 的运行时校验。
3. Responses API provider 与错误分类。
4. 费用、缓存、超时和离线降级。
5. 多视角截图与 Gold Set。

在以上工作完成前，不启用阶段二按钮，也不讨论高度自主拆件。
