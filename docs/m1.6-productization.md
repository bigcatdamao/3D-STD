# 3D Studio · M1.6 AI 拆件助手产品化路线

| 项 | 内容 |
|---|---|
| 版本 | v0.6 |
| 日期 | 2026-07-23 |
| 目标 | 先建立可信、可解释、只读的拆件决策体验，再逐步开放预览和确认后执行 |
| 当前批次 | M1.7.9 主动拆件入口与工作台；M1.6 Gold Set 暂缓补齐 |

## 1. 产品定位

M1.6 不是“一键自动拆件”，而是把已有场景、打印空间和网格检查结果组织成用户能够判断的方案。阶段一 AI 分析链路已完成技术验证，但实际产品评测发现文字建议缺少直接作用，因此 M1.7.9 把它从当前主界面撤下，保留 API、Schema 与测试作为历史实验能力。当前产品以用户主动触发的确定性拆件工作台为主；未来若恢复 Agent，只能辅助解释或编排预览，不能替代清晰的手动按钮。第三阶段仍必须在用户确认后执行，并具备原始副本、历史记录和撤销保障。

## 2. 版本拆分

### M1.6.1 · 只读体验原型（已完成）

- 右侧检查器新增“AI 拆件”Tab，打印检查完成页增加入口。
- 自动汇总对象数、世界尺寸、打印床、检查状态和现有拓扑问题。
- 提供拆件目标、打印方式和四类优先目标。
- 本地 Mock 返回“是否拆件、原因、建议件数、候选方案、优缺点、风险、下一步”的完整产品结构。
- 缺失薄壁、局部过悬、候选切面和多视角证据时如实显示能力限制。
- 场景变化后结果立即过期；分析不修改场景、不写操作历史。

### M1.6.2 · 真实 Responses API 闭环（已完成）

- 新增后端 `/api/agent/split-analysis`，`OPENAI_API_KEY` 仅允许来自 Worker Secret。
- 前端生成前、右、顶、轴测 4 张 384px JPEG 低细节视图；不影响实时视口。
- Responses API 使用 `store:false`、明确 `detail: low` 和严格输出 JSON Schema。
- 服务端实现输入限额、75 秒超时、refusal、incomplete、输出校验和脱敏错误分类；延迟达标仍以 Gold Set 的 p95 ≤ 45 秒作为优化门槛。
- 使用独立 Durable Object 实例执行单访客日限额、全站熔断和失败退款，不混用 Tripo credits。
- UI 明确显示“AI 分析”或“本地降级建议”，离线和服务失败不伪装成模型结果。
- 当前未实现快照缓存、usage token 账本和请求幂等存储；这些随真实流量与评测数据决定是否进入 M1.6.3。

### M1.6.2.1 · AIHubMix Provider 适配（已完成）

- Worker 仅允许 `openai` / `aihubmix` 两个固定 Provider，浏览器和环境变量都不能注入任意上游 URL。
- AIHubMix 使用独立 Worker Secret `AIHUBMIX_API_KEY`；`OPENAI_API_KEY` 保留为官方通道回滚，不会自动混用。
- 继续使用同一 Responses 请求、`store:false`、严格输出 Schema、超时、配额、退款和本地降级机制。
- UI 在提交前说明结构化数据与多视角截图会交给后台模型服务；成功后明确显示实际 Provider、模型和视角数。

### M1.6.3 · 质量评测与成本校准

- 建立 20–30 个代表模型的 Gold Set，评估需要拆件判断、方案可用性、幻觉率、延迟和成本。
- 对照低/中推理强度和候选模型；根据评测校准图片尺寸、输出上限、日限额和熔断值。
- 评估是否需要快照缓存、usage 账本和幂等存储，不为零流量提前增加状态复杂度。
- 达到预设门槛后再关闭阶段一；未达标时不进入预览工具开发。
- 第一批已实现结构化步骤操作卡：定位检查问题、切换打印检查和重新检查由本地确定性代码执行；切割与朝向工具继续锁定为阶段二。
- 第一批已实现响应耗时与 token 遥测、Gold Set JSON 契约、5 个种子场景、自动评分器和人工评分规则；仍需补齐 19 个已授权真实模型。

### M1.7 · 确定性修复工作台与只读预览（已完成并线上验收）

- 第一批先解决“检查能发现问题但不能修”的断点：非水密/退化错误可生成确定性修复预览。
- 修复预览进入独立临时层，场景变化即失效；确认前不改场景、不写历史。
- 确认后以派生资产替换当前实例引用，原资产保留，单条历史支持撤销/重做，并自动复检。
- `inspect_model` 可继续复用打印检查；切割候选、平面切割预览、方案比较和朝向分析后置到修复闭环稳定之后。

### M1.7.1 · 深度网格诊断与修复差异（开发完成）

- 检查 Worker 新增自交、内部壳、孤立碎片只读证据，并把连通壳数量写入信息清单。
- 自交扫描设置明确预算；部分扫描不会伪装成完整通过。
- 复杂问题只显示“只读诊断”，不提供自动修复入口；Agent 权限不变。
- 修复预览增加叠加/仅看变化切换，以绿色/红色区分新增和删除面。

### M1.7.2 · 自交局部证据定位（开发完成）

- 深度检查保留有限数量的确定自交三角形对和原始面片编号，证据仍随资产级缓存复用。
- 打印检查条目增加红/蓝面片图例、上一组/下一组和定位动作；视口只显示当前命中对并聚焦局部包围盒。
- 浏览与聚焦是只读 UI 状态，不写操作历史；完整模型红壳在自交证据模式下隐藏，避免遮挡局部证据。
- 局部三角形坐标不进入大模型请求，Agent 权限和费用边界不变。

### M1.7.3 · 连通壳拆件只读预览（开发完成）

- 检查 Worker 在连通壳统计之外返回受预算保护的原始面片索引、壳类型、面数、闭合状态和局部包围盒。
- 尺寸信息条目增加“预览拆成 N 件”；首次进入展示整体分色，上一件/下一件再带观察余量聚焦局部壳。
- 主体、独立壳、疑似内部壳和疑似碎片沿用确定性诊断分类；预览不会创建零件、修改原几何或写历史。
- 预览最多展示前 24 个壳、总计 120,000 个面；预算触顶时明确显示抽样预览，完整壳数量仍保持准确。
- 新增 `?qa=component-preview` 稳定验收入口；Agent 继续只接收连通壳数量摘要，不接收面片索引。

### M1.7.4 · 平面切割候选只读预览（开发完成）

- 单一连通壳超出打印床时，本地纯函数沿世界 X/Y/Z 中线生成三个候选，并估算两侧包围盒尺寸、床适配、剩余超限轴和切面面积。
- 视口以 GPU clipping plane 呈现青/紫两侧和琥珀切面；不计算真实交线、截面封口或派生网格，因此 UI 始终显示“包围盒估算”。
- 候选存在独立 PreviewStore，绑定 `editVersion` 与床配置；切换、关闭和相机聚焦不写历史，场景/床变化后旧结论立即失效。
- AI 建议页在有新鲜单壳超床证据时可打开这套本地候选，但 Agent 仍不能执行切割，阶段三写权限没有开放。
- 新增 `?qa=plane-cut-preview`；X 中切后两侧均可进入 256mm 床，Y/Z 仍超床，完整流程历史保持 `0/0`。

### M1.7.5 · 可调平面切割位置（开发完成）

- 每个 X/Y/Z 候选增加 10%–90% 连续位置和 25% / 50% / 75% 快捷点，调整时 A/B 尺寸、床适配和切面同步刷新。
- 按当前打印床计算“两侧都可放入”的安全区间；不存在单切可行区间时直接说明，不让用户盲拖。
- 三个轴分别保留自己的临时位置，便于来回比较；预览仍不改场景、不写历史、不调用模型。
- 当前只是 AABB 和 GPU 裁剪结果，尚未加入真实截面、自然接缝、薄颈、强度、封口和派生零件。

### M1.7.6 · 真实平面截面证据（开发完成）

- 按实例世界变换计算三角形与当前切面的真实交线，视口以亮黄色叠加轮廓。
- 完整简单闭环显示环数、截线数、周长和净面积；开链、分叉、共面歧义、空截面与部分扫描不输出面积。
- 默认扫描上限 120,000 面、截线上限 24,000 条；高面数模型只显示预算内证据并明确降级。
- 滑杆先即时更新切面和 A/B 尺寸，停止约 90ms 后再扫描真实截面；快捷点保持立即计算。
- 截面证据仍不是切割、封口或派生零件，Agent 不获得顶点数据和写权限。

### M1.7.7 · 几何低风险接缝推荐（开发完成）

- 用户显式触发世界 X/Y/Z 各 9 个位置的 Worker 扫描，最多给出 3 个完整闭合且双侧入床的接缝候选。
- 评分综合局部薄颈、截面面积、轮廓复杂度、截线密度和零件比例；每张卡说明理由与风险，同轴近似重复位置会被去重。
- 点击推荐直接跳转现有可调切面与黄色真实轮廓，滑杆下同步显示推荐标记；全过程仍为只读 PreviewStore。
- Worker 成功后复用资产几何缓存，取消/异常/45 秒超时立即终止并在重试时重传；批量结果不把截线坐标送回主线程。
- 这不是自然语言 Agent、自主工具循环、结构强度验证或语义部位识别，也不生成可导出的切后零件。

### M1.7.8 · 表面自适应真实切割预览（开发完成）

- 轴向推荐只作为引导中心；用户设置 ±4%–±25% 表面吸附范围后，本地 Worker 在源模型面邻接图上搜索沿现有网格边的闭合接缝。
- 接缝代价偏好较短路径、明显折角与较低三角形密度，并惩罚远离引导中心的边；QA 收腰件的引导中心 X=10mm，实际接缝自动移动到 X=20mm。
- 成功结果是两份真实内存临时网格：保留各自源三角形、双侧反向封口，并验证开放边为 0；画布不再用固定水平/轴向 clipping plane 充当最终能力。
- 只支持单一连通水密边流形、单闭合接缝和最多 80,000 面；非流形、多壳、分叉、多环、预算或封口失败全部拒绝。
- 封口仍是单中心扇形，不验证高凹截面自交、强度、装配与可制造性；结果不创建场景对象、不写历史、不导出，Agent 写权限不变。

### M1.7.9 · 主动拆件入口与工作台（开发完成）

- 画布左上固定增加“✂ 拆件”，面向当前单个选中对象；无选择或多选时给出明确下一步。
- 点击后自动打开打印检查；尺寸证据缺失或过期时自动运行检查，完成后按实例定位并展开拆件工作台。
- 工作台改为“1 选对象 → 2 调引导 → 3 看切割”，X/Y/Z 由显式按钮选择；不再依赖超出打印床的隐藏错误条目。
- 当前 UI 移除“AI 拆件”Tab 和检查底部文字建议 CTA；旧 `split` 布局偏好自动迁移到 `check`。
- M1.7.8 的表面搜索与 A/B 封口内核保持不变；M1.7.9 解决的是可发现性和操作闭环，仍不开放写入或导出。

### M1.8 · 确认后执行

- 在 copy-on-write、审批票据、原子历史和撤销全部就绪后，再开放切割、合并、定位销和装配验证。
- 每次写操作必须遵循“预览 → 用户确认 → 执行 → 验证 → 历史入栈”。

## 3. M1.6.1 历史关闭标准（当前 UI 已撤下）

- 用户能从打印检查自然进入 AI 拆件分析。
- 结果字段完整，并能比较候选方案。
- 不把整件离床误称为局部过悬，不把未检测能力显示为通过。
- 不生成伪精确坐标或伪切割预览。
- 分析前后场景版本、对象数和历史长度不发生变化。
- 自动化测试、类型检查、生产构建和真实浏览器验收通过。

## 4. M1.6.2 历史关闭标准（API 保留、当前 UI 已撤下）

- Worker Secret 未配置时 fail-closed，并自动给出明确本地降级结果。
- 4 张离屏截图和结构化场景快照能进入服务端；浏览器看不到服务端 API Key。
- 严格 Schema 成功返回 2–3 套方案；拒绝、超时、上游失败和坏输出均分类收敛。
- 失败退款、单访客日限额和独立全站熔断通过自动化验证。
- 分析前后场景版本、对象数和历史长度不发生变化。
- 类型检查、全量自动化、生产构建和外部 Chrome 验收通过。

## 5. 下一步优先级

1. 用真实角色、机械件和有机模型从 M1.7.9 主入口完成任务，对照人工切线校准引导带宽、折角、密度和边长权重，记录成功、拒绝和误导样本。
2. 把中心扇形升级为投影/约束三角化与局部重网格，补高凹和非平面接缝的自交验证；未完成前不把临时封口称为生产零件。
3. 设计“确认后把验证通过的 A/B 网格写成派生零件”的 copy-on-write、lineage、命名、原子历史和撤销契约。
4. M1.6 Gold Set 保留为 AI 决策质量专项，在真实使用量或工具编排需要时恢复。

在修复与切割预览形成稳定闭环前，不启用高度自主拆件。
