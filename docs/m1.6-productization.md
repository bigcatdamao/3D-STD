# 3D Studio · M1.6 AI 拆件助手产品化路线

| 项 | 内容 |
|---|---|
| 版本 | v0.1 |
| 日期 | 2026-07-21 |
| 目标 | 先建立可信、可解释、只读的拆件决策体验，再逐步开放预览和确认后执行 |
| 当前批次 | M1.6.2 Responses API 只读闭环已完成，待线上 Secret 与真实样本验收 |

## 1. 产品定位

M1.6 不是“一键自动拆件”，而是把已有场景、打印空间和网格检查结果组织成用户能够判断的方案。第一阶段只做分析和建议；第二阶段才提供可丢弃的切割预览；第三阶段必须在用户确认后执行，并具备原始副本、历史记录和撤销保障。

## 2. 版本拆分

### M1.6.1 · 只读体验原型（已完成）

- 右侧检查器新增“AI 拆件”Tab，打印检查完成页增加入口。
- 自动汇总对象数、世界尺寸、打印床、检查状态和现有拓扑问题。
- 提供拆件目标、打印方式和四类优先目标。
- 本地 Mock 返回“是否拆件、原因、建议件数、候选方案、优缺点、风险、下一步”的完整产品结构。
- 缺失薄壁、局部过悬、候选切面和多视角证据时如实显示能力限制。
- 场景变化后结果立即过期；分析不修改场景、不写操作历史。

### M1.6.2 · 真实 Responses API 闭环（已完成）

- 新增后端 `/api/agent/split-analysis`，`OPENAI_API_KEY` 仅允许来自 Worker Secret。
- 前端生成前、右、顶、轴测 4 张 384px JPEG 低细节视图；不影响实时视口。
- Responses API 使用 `store:false`、明确 `detail: low` 和严格输出 JSON Schema。
- 服务端实现输入限额、45 秒超时、refusal、incomplete、输出校验和脱敏错误分类。
- 使用独立 Durable Object 实例执行单访客日限额、全站熔断和失败退款，不混用 Tripo credits。
- UI 明确显示“AI 分析”或“本地降级建议”，离线和服务失败不伪装成模型结果。
- 当前未实现快照缓存、usage token 账本和请求幂等存储；这些随真实流量与评测数据决定是否进入 M1.6.3。

### M1.6.3 · 质量评测与成本校准

- 建立 20–30 个代表模型的 Gold Set，评估需要拆件判断、方案可用性、幻觉率、延迟和成本。
- 对照低/中推理强度和候选模型；根据评测校准图片尺寸、输出上限、日限额和熔断值。
- 评估是否需要快照缓存、usage 账本和幂等存储，不为零流量提前增加状态复杂度。
- 达到预设门槛后再关闭阶段一；未达标时不进入预览工具开发。

### M1.7 · 只读工具与预览

- 实现 `inspect_model`、`find_cut_candidates`、`preview_plane_cut`、`compare_split_schemes`、`analyze_print_orientation`。
- 预览进入独立临时层，场景变化即失效，不能写入原始模型和历史。

### M1.8 · 确认后执行

- 在 copy-on-write、审批票据、原子历史和撤销全部就绪后，再开放切割、合并、定位销和装配验证。
- 每次写操作必须遵循“预览 → 用户确认 → 执行 → 验证 → 历史入栈”。

## 3. M1.6.1 关闭标准

- 用户能从打印检查自然进入 AI 拆件分析。
- 结果字段完整，并能比较候选方案。
- 不把整件离床误称为局部过悬，不把未检测能力显示为通过。
- 不生成伪精确坐标或伪切割预览。
- 分析前后场景版本、对象数和历史长度不发生变化。
- 自动化测试、类型检查、生产构建和真实浏览器验收通过。

## 4. M1.6.2 关闭标准

- Worker Secret 未配置时 fail-closed，并自动给出明确本地降级结果。
- 4 张离屏截图和结构化场景快照能进入服务端；浏览器看不到服务端 API Key。
- 严格 Schema 成功返回 2–3 套方案；拒绝、超时、上游失败和坏输出均分类收敛。
- 失败退款、单访客日限额和独立全站熔断通过自动化验证。
- 分析前后场景版本、对象数和历史长度不发生变化。
- 类型检查、全量自动化、生产构建和外部 Chrome 验收通过。

## 5. 下一步优先级

1. 在 Cloudflare 配置/核对 `OPENAI_API_KEY`，完成一次真实多模态结果验收。
2. 建立 20–30 个模型的 Gold Set 和人工基准答案。
3. 记录判断准确性、方案可用性、幻觉率、延迟和单次成本。
4. 用评测结果决定模型、推理强度、图片尺寸、输出上限和缓存策略。
5. 阶段一达到门槛后再进入 M1.7 只读几何工具与预览。

在以上工作完成前，不启用阶段二按钮，也不讨论高度自主拆件。
