## 产品目标

把 M1.7.4 的“只能对半看一眼”推进成可比较的决策工具：用户可以主动移动切面，立即知道两侧尺寸和床适配是否仍成立，并看见当前轴真正可用的位置范围，而不是只得到一段文字建议。

## 新增体验

1. X/Y/Z 每个候选增加 10%–90% 连续滑杆，以及 25% / 50% / 75% 快捷位置。
2. 调整时视口切面、A/B 包围盒尺寸、床适配、推荐分和说明实时同步。
3. 卡片显示切面距 A 端的实际毫米数，帮助用户把归一化百分比对应到模型尺度。
4. 当前打印床下若存在“两侧都可放入”的位置范围，直接显示可放入区间；不存在时明确提示一次轴向切割无法解决。
5. X/Y/Z 分别保存自己的临时偏移，来回切换不会抹掉用户在另一轴上的比较位置。
6. 滑杆、快捷点和候选切换继续保持历史 `0/0`，不调用 AI、不修改模型。

## 确定性计算口径

- `evaluatePlaneCutCandidate` 接收源世界包围盒、当前打印床、轴和请求位置；请求位置统一限制为 10%–90%。
- A/B 结果由同一世界 AABB 在切面处连续拆分，另两轴范围保持不变。
- 可放入区间只在另两轴已经入床、且目标轴总长不超过床长两倍时成立；区间边界为 `1 - 床长 / 总长` 到 `床长 / 总长`。
- 300mm 长模型放入 256mm 床时，X 轴理论区间约 14.67%–85.33%，UI 向内取整显示为 `15%–85%`。
- 推荐分仍限制在 0–100，并加入位置平衡项；评分只用于候选比较，不是打印成功概率。

## 几何与权限边界

- 当前结果仍是世界包围盒估算，不是精确切后体积或真实截面。
- 视口继续使用 GPU clipping plane；不做三角形求交、轮廓闭合、孔环识别或封口。
- 不生成派生资产、不增加对象、不导出 A/B STL，也不写操作历史。
- PreviewStore 绑定源 `editVersion` 与打印床；编辑场景、改变床尺寸或删除实例后旧预览失效。
- `apply_plane_cut`、`merge_parts`、`add_alignment_pin`、`validate_parts` 和 `validate_assembly` 仍未开放。
- Agent 没有获得工具循环或写权限；本版本只增强用户显式操作的本地只读预览。
