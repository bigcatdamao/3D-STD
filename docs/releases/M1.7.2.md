## 产品目标

把“检测到 N 组自交”的抽象错误升级为用户可以看见、核对和逐条浏览的局部证据，同时坚持只读诊断：不自动修复复杂拓扑，不修改原模型，不写操作历史。

## 新增体验

1. 自交检测保留原始面片编号和两片三角形局部坐标，UI 显示“面 #A × #B”。
2. 错误条目增加红色面 A、蓝色面 B 图例，以及上一组、下一组和定位操作。
3. 命中对在视口以穿透面、亮边和中心标签显示；当前证据可在实体内部被看见。
4. 点击条目或浏览按钮会把相机聚焦到当前两片三角形的世界包围盒，而不是整个模型。
5. 自交证据模式隐藏全模型红壳，避免大面积错误色遮住真正的局部证据。
6. 浏览索引、选择和相机都属于只读 UI 状态，历史、资产和模型几何保持不变。
7. 新增隐藏 QA 地址 `/?qa=self-intersection`：三片三角形形成两组稳定命中，自动打开检查面板并运行检查。

## 数据与性能边界

- 自交总命中仍沿用 M1.7.1 的 200 组上限；可视证据独立限制为前 24 组。
- 面片编号来自原始三角面顺序，使用 1-based 展示，便于和外部网格工具核对。
- 证据挂在资产级健康元数据上；同一资产的多个实例复用检测结果，仅在呈现时应用各自 TRS。
- Agent 输入继续只发送自交数量、扫描完整性等摘要，不发送局部三角形坐标。
- 当前显示的是“发生相交的三角形对”，不计算精确交线、不自动删除或重拓扑。
- 部分扫描只代表已覆盖范围；未命中仍不等于不存在自交。

## 本地 Chrome 验收

- QA 场景自动检查得到 2 个错误、0 个警告；自交项为 2 组命中。
- 第一组显示“证据 1/2 · 面 #1 × #2”，第二组显示“证据 2/2 · 面 #1 × #3”。
- 点击下一组后视口标签、红蓝面片和局部相机同步切换；继续点击会循环回第一组。
- 全流程场景为 1 个对象、3 个面，历史保持 `0/0`。
- 常规五对象示例未加入 QA 夹具，M1.6 Agent Gold Set 仍保持原五对象口径。

## 手动验收清单

1. 本地运行后打开 `http://127.0.0.1:4173/?qa=self-intersection`；线上部署后在生产地址追加同一查询参数。
2. 确认打印检查自动完成，自交条目显示 2 组证据和红蓝图例。
3. 点击下一组，确认编号从 `#1 × #2` 变为 `#1 × #3`，视口和相机同步变化。
4. 再点下一组，确认循环回第一组；点上一组也能反向循环。
5. 确认历史始终为 `0/0`，资产和对象数量不变。
6. 可选：手动导入 `docs/test-assets/self-intersection-pair.stl`，验证真实导入链得到同类结果。
