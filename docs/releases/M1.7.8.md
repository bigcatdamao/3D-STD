## 产品结论

M1.7.8 首次越过了“固定水平/轴向平面视觉裁剪”的能力上限：轴向位置只作为搜索引导，真实接缝可以在用户给定带宽内沿模型表面移动，吸附到更短收腰或明显折痕，并输出两份拓扑封口的 A/B 临时三角网格。

但这仍不是可以直接生产和导出的最终切割。接缝只能沿现有网格边，封口是单中心扇形；本版只验证拓扑开放边为 0，没有完成凹环/非平面自交、局部重网格、属性继承、结构强度、装配公差和可制造性验证。

## 真实表面接缝内核

1. 把实例局部顶点按 TRS 转为世界坐标，焊接重复坐标并拒绝无效或退化三角形。
2. 验证每条焊接边恰好关联两个面，并验证面邻接图只有一个连通分量；开口、非流形和多壳源模型直接拒绝。
3. 以引导轴/位置和用户吸附带宽建立两侧硬种子；带宽覆盖一侧、无法形成 A/B 种子时拒绝。
4. 在面邻接图上运行 s-t 最小割。边代价综合世界边长、距引导中心的平方惩罚、相邻面折角和局部三角形密度。
5. 分区边界必须形成一个无分叉有向闭合环；分叉、多环、空边界或超过 12,000 边预算时拒绝。

## A/B 网格与验证

- 按面标签把真实源三角形复制到 A/B 两侧，不用 GPU 裁剪结果冒充几何。
- 对同一闭合接缝生成相反绕序的双侧中心扇形封口。
- 对两份非索引网格重新焊接计边；仅当 A/B 开放边都为 0 时返回 ready。
- 返回真实尺寸、源面数、封口面数、开放边、接缝点数/长度、相对引导偏移、表面跨度和平均折角。
- 画布显示青/紫真实临时网格和亮色表面接缝；右栏不再在成功态重复显示旧包围盒代理。

## Worker 与状态收口

- `surface-cut.worker.ts` 独立执行图构建、最小割、封口和验证，避免阻塞主线程。
- 首轮 Transfer 资产 position/index；同资产成功后只传资产 ID、TRS、引导位置和带宽。
- 取消、Worker 异常或 60 秒超时立即终止 Worker 并清除缓存标记；重试会重新发送几何。
- 结果绑定实例 ID、`editVersion`、轴、引导位置和带宽；条件变化后迟到结果不能进入 UI。
- 整个预览不修改 `SceneDocument`、资产或历史。

## 能力边界

- 只支持单一连通、水密、边流形、无退化面的三角网格。
- 源面上限 80,000；接缝边上限 12,000；不对超预算模型抽样切割。
- 只支持一个闭合接缝环，不支持分叉、多环、多个独立部件或语义拆件区域。
- 接缝沿已有网格边移动，不是任意连续曲面刀线；低模的结果会明显受拓扑分辨率限制。
- 中心扇形封口对高凹、带孔或强非平面环不够鲁棒；开放边为 0 只证明拓扑闭合，不证明无几何自交。
- 不创建场景零件、不写派生资产 lineage、不进历史、不导出 A/B STL，不开放 Agent 写工具。

## 下一步建议

1. 用角色、机甲、有机外壳和低模各 3–5 个真实模型，对比人工选择的接缝，校准带宽和图割权重。
2. 增加用户表面点选/笔刷约束，让“要拆哪里”不再只能由轴向引导表达。
3. 把中心扇形升级为约束三角化与局部重网格，并验证凹环/非平面环自交、法线、体积和属性。
4. 几何验证稳定后，再做 copy-on-write 派生 A/B 资产、lineage、用户确认、单条原子历史和撤销；在此之前不开放应用切割。
