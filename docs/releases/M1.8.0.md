## 为什么做这个版本

M1.7.4–M1.7.7 建立了轴向切面候选、位置调整、真实截面证据和低风险位置推荐；M1.7.8 能在内存里预览沿源网格表面移动的真实 A/B 临时网格；M1.7.9 解决了拆件入口不可发现的问题。但这些版本仍然没有把用户确认后的结果写成两个场景模型。

M1.8.0 先完成边界清晰、可稳定验证的平面切割：用户像 Blender 一样直接控制切割平面，明确点击确认，程序真实生成两个独立派生模型；撤销可以完整恢复原对象。本版不增加榫卯，也不把曲面自适应预览包装成已经可生产写入的能力。

## 用户操作路径

1. 在场景树或画布中选中一个对象。
2. 点击画布左上“✂ 拆件”。
3. 用 W / E / R 切换移动、旋转、大小模式；拖动画布中的 XYZ 箭头、旋转环和缩放手柄，或在右栏调整位置、旋转与显示宽高。
4. 可用右栏 X / Y / Z 快捷方向把切面恢复为正交方向。
5. 点击“确认切割”；计算期间可取消，源对象在结果成功前不会变化。
6. 成功后场景树出现“原名 · A”和“原名 · B”两个独立模型，切割框退出。
7. 在历史页点击撤销恢复原对象；重做再次恢复 A/B。

## 几何实现

- `plane-split-core.ts` 支持索引和非索引三角网格，在源资产局部空间对任意无限平面逐三角形裁切。
- 交叉三角形两侧分别补新顶点，截线按容差焊接成闭合环；支持多个轮廓和嵌套孔洞。
- 截面投影到平面二维基底，使用 `THREE.ShapeUtils.triangulateShape` 生成切口三角形；A/B 使用相反绕序与法线封口。
- 平面恰好沿已有流形边通过时可继续构成闭环；完全共面面片造成归属歧义时拒绝写入。
- 默认最多处理 2,000,000 个源面；Worker 超时为 60 秒，取消或超时会终止当前计算。

## 场景、资产与历史契约

- 世界切面先按实例位置、XYZ 欧拉旋转和非等比缩放转换到源资产局部空间；原资产几何只复制读取。
- 成功结果注册为两个新的派生几何和资产，继承源实例的父级、顺序、TRS、显示状态与基础材质属性。
- `splitInstanceWithDerivedParts` 在一个历史命令中删除源实例并插入 A/B；历史标题为“✂️ 平面切割 · 对象名”。
- 撤销同时移除 A/B 实例和派生资产并恢复源实例；重做再次恢复 A/B。原始资产从不被原位修改。
- 失败发生在历史写入之前，因此不会留下半个模型、空资产或无法撤销的中间态。

## UI 与交互

- 切割模式使用专用半透明青/紫平面、琥珀框与十字线，不与普通对象 Gizmo 混用。
- 画布提供 XYZ 移动箭头、XYZ 旋转环和大小手柄；右栏提供 W/E/R 模式、X/Y/Z 方向预设、位置 XYZ、旋转 XYZ、宽/高。
- 切割模式下暂停普通选择、Delete 删除和对象 TransformControls，避免误操作；Esc 或“取消”安全退出。
- 框宽高只控制显示范围，实际切割使用无限数学平面，避免模型超出框后出现未切干净的误解。
- 场景 `editVersion` 或源实例发生变化后，待确认结果立即标为过期，不能把旧切面写到新场景。

## 已知边界

- 本版没有榫卯、定位销、合并或装配验证。
- M1.7.8 曲面自适应切割仍是临时预览，没有接入本版确认写入链。
- 平面大小是视觉框，不是有限切割盒；真实裁切平面无限延伸。
- 结果重算法线并保留几何与基础材质外观，但不保证 UV、切线、顶点色、蒙皮、变形目标或多材质组继承。
- 不修复源模型已有的自交、内部壳、非流形或开放边；源模型缺陷可能使切割被拒绝，成功也不等于已经通过所有打印检查。
- 双侧封口的拓扑闭合不等于结构强度、装配公差、无自交或最终可制造性验证。
- 多面共面、截面开放/分叉、封口三角化失败、空零件、无相交和超预算均 fail-closed；失败时原对象不变。

## 下一步建议

1. 先用机甲、角色、有机件、低模和带多个截面轮廓的模型各 3–5 个做真实样本验收。
2. 补 UV、法线、切线、材质组和蒙皮模型的继承/拒绝契约，再扩大可用资产范围。
3. 升级 M1.7.8 的非平面封口与自交验证后，再把曲面自适应结果接入同一 copy-on-write 与原子历史链。
4. 最后再单独设计定位销/榫卯，不能把它和基础裁切耦合成一次不可撤销操作。
