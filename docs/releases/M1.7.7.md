## 产品目标

把 M1.7.6 的“用户逐个拖动切面看证据”推进为“系统先扫描一组可解释的几何候选，再让用户比较和预览”。本版解决的是轴向切面搜索效率，不宣称理解角色部位、机械结构、装配语义或受力，也不开放自动切割。

## 新增体验

1. 平面切割候选卡增加“几何低风险接缝”，用户点击后扫描世界 X/Y/Z 各 10%–90% 共 27 个位置。
2. 扫描过程显示完成数与进度条，并可随时取消；失败或超时有明确重试入口。
3. 完成后最多显示 3 个推荐，包含轴、位置、0–100 分、最多 3 条理由和至少 1 条风险边界。
4. 当前轴的推荐同步显示在滑杆下方；点击推荐卡或标记，会直接跳到目标切面并显示完整黄色真实轮廓。
5. UI 固定说明“评分是几何代理，不识别角色脸部、机械关节、受力或装配语义”。

## 候选准入与评分口径

- 固定采样世界 X/Y/Z 三轴，每轴 10%、20%……90% 共 9 个位置，总计 27 个样本。
- 只有切后 A/B 两侧都能放入当前打印床，并且真实截面为完整、无歧义闭合环、面积大于 0 的样本才进入排名。
- 总分权重：双侧入床 35；与相邻切片比较得到的局部薄颈 20；相对小截面面积 16；轮廓紧凑度与少环 14；低截线密度 10；零件比例均衡 5。
- 同轴两个推荐位置小于 15% 时视为近似重复，只保留分数更高者。
- “截线密度”只表示单位周长上的网格求交线数量，是细节密度代理；“紧凑度”也不是结构强度或连接可靠性。

## Worker、缓存与失败收口

- 批量扫描在独立 `seam-scan.worker.ts` 中运行，复用 M1.7.6 的世界三角形-平面真实求交内核。
- 首次扫描把当前资产的 position/index 紧凑副本以 Transferable 发送给 Worker；成功后同资产再次扫描只传 `assetId`、实例 TRS 和切面列表。
- Worker 只回传 27 个无线段坐标的截面摘要，完整黄色轮廓只在用户点击具体推荐后由现有单点预览计算。
- 取消、Worker 异常或 45 秒超时会立即终止 Worker 并清空缓存标记；下一次重试重新发送几何，不继续使用可能不完整的状态。
- 场景编辑、打印床变化、实例删除或预览切换后，旧扫描在下一次进度消息处终止，迟到结果不能覆盖新状态。

## 几何与权限边界

- 推荐仍限定世界 X/Y/Z 轴向平面，不搜索任意法向、曲面接缝或多平面组合。
- 当前没有可靠曲率、局部平坦度、隐藏接缝、强度、支撑、连接结构或装配可达性结论。
- 不分类并重建切后三角形，不生成边界新顶点、不封口、不创建派生资产、不导出 A/B STL。
- 扫描、推荐跳转、相机和关闭预览只改临时 PreviewStore，不增加 `SceneDocument.editVersion`，不写操作历史。
- 该 Worker 是浏览器本地确定性工具，不是 Responses API 或 Agent 自主工具循环；模型仍无 `apply_plane_cut` 或其他写权限。

## 下一步建议

1. 先用 5–10 个真实复杂模型做推荐有效性校准，记录推荐位置、人工选择和误导原因，不立即开放写操作。
2. 评估曲率突变与局部平坦度是否能提供稳定增益；没有对照证据前不使用“自然接缝”标签。
3. 若要进入真实切割，先补新顶点拓扑、孔环方向、封口三角化、水密验证、派生资产 lineage 和原子撤销。
4. 继续保持用户确认门；M1.7.7 的推荐不能直接升级成“应用切割”。
