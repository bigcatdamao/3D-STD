## 产品目标

把检查结果里的“3 个连通壳”从一行统计升级为用户可直接理解的候选零件预览：先看整体分色，再逐件定位；本版本只解释现有几何结构，不直接生成新零件。

## 新增体验

1. 含多个连通壳的尺寸信息条目增加“连通壳拆件预览”和“预览拆成 N 件”入口。
2. 首次进入展示整个对象，主体、独立壳、内部壳和疑似碎片使用循环色板区分。
3. 上一件、下一件和定位操作会聚焦当前壳，并保留约 60% 观察余量以呈现相邻关系。
4. 视口标签显示“零件 N/M、类型、面数”，当前壳增加穿透高亮和局部亮框。
5. 浏览索引、选择与相机均属于只读 UI 状态，不创建零件、不修改原模型、不写历史。
6. 新增隐藏 QA 地址 `/?qa=component-preview`：一个资产内的三个封闭立方壳自动检查并进入预览。

## 数据与性能边界

- 连通壳完整分析仍限制在 250,000 面以内。
- 可视预览最多返回前 24 个壳、总计 120,000 个面索引；超限时按壳公平抽样并显示“预算内抽样预览”。
- 面索引只在本地检查 Worker 与视口间传递；Agent 仍只接收 `connectedComponents` 数量摘要。
- 预览复用资产级健康缓存；同资产多实例仅在呈现时应用各自 TRS。
- 当前没有创建派生资产、自动命名、合并/拆分确认或跨壳装配关系。

## 手动验收清单

1. 打开 `http://127.0.0.1:4173/?qa=component-preview`；线上部署后在生产地址追加同一查询参数。
2. 确认打印检查自动完成并显示“3 个候选零件”“完整预览”。
3. 确认首屏同时看到三个不同颜色的壳，而不是直接贴近第一个壳。
4. 连续点击下一件，确认 1→2→3→1 循环，标签、亮框和相机同步变化。
5. 确认历史始终为 `0/0`，对象仍为 1 个，未生成任何新资产或新实例。

## 下一步建议
