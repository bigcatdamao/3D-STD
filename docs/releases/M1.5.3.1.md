# 3D Studio M1.5.3.1 版本记录

| 项 | 内容 |
|---|---|

| 日期 | 2026-07-21 |
| 上一版本 | M1.5.3 图生模型 |
| 范围 | 视口优先布局、中央 AI 创作面板、右侧历史 Tab |
| 状态 | 开发完成，待部署后手动验收 |

## 产品变化

1. 删除固定底部 AI/历史工作区，3D 画布从顶栏下方一直延伸到窗口底部。
2. 空场景中央直接显示 AI 创作面板，文字、单图和三图入口不再与 3D 画面争夺纵向空间。
3. 创作面板增加独立背景层，使前景输入区与后方 3D 空间层级清楚；场景已有模型时可从顶栏 ✦ 再次打开并关闭。
4. 历史记录进入右侧栏第三个 Tab，采用上旧下新的纵向时间线，保留撤销、重做、点击跳转、悬停高亮和预览冻结。
5. 右侧 Tab 直接显示历史当前位置 `N/M`，打印检查问题数与历史进度都无需先进入页面才能判断。

## 布局规则

- 桌面宽屏：左栏、全高视口和右栏三列并列。
- 首次空场景：创作面板不可误关，避免用户回到没有下一步的空白画布。
- 已有模型：创作面板为可关闭覆盖层，点击背景或右上角关闭即可返回编辑。
- 约 1366px：右侧栏使用覆盖抽屉；AI 面板会自动向左避让，避免输入区被遮挡。
- 更窄空间：生成方式切换为横排，输入区和提交区改为纵向布局。

## 不变的功能语义

- AI 文字、单图和多图仍复用同一生成状态机、配额、Turnstile、失败处理和接受链路。
- 图片格式、大小、数量及三视角规则不变。
- 历史仍由同一 `HistoryManager` 驱动；布局切换不产生历史记录。
- 场景文档、资产/实例分离、打印检查和 STL 导出语义均未修改。

## 验证记录

- TypeScript 全项目检查通过。
- 全量自动化：34 个测试文件、374 项测试通过。
- Chrome 实际页面检查通过：文字、单图、三图中央布局，以及右侧历史 Tab 均无溢出。
- 1366×768 响应式检查通过：创作面板与右侧历史栏同时打开时无重叠。
- 浏览器控制台无 error 或 warning。

## 部署后手测

1. 空场景确认底部无固定面板，画布高度明显增加，中央创作面板完整可见。
2. 分别切换文字、单图、多图，确认输入区尺寸稳定，生成按钮不被遮挡。
3. 打开示例或导入模型，确认创作面板自动退出；点击顶栏 ✦ 可再次打开并关闭。
4. 在右侧切换“属性 / 打印检查 / 历史”，确认三个页面互不挤压。
5. 执行移动、删除等操作，确认历史从上到下新增，撤销、重做、点击跳转和悬停高亮正常。
