## 本版新增

1. 右侧检查器扩展为“属性 / 打印检查 / AI 拆件 / 历史”四个 Tab。
2. 打印检查结果页增加 AI 拆件入口，让用户从质量问题自然进入决策流程。
3. 自动整理当前对象、打印床、世界尺寸、检查新鲜度、非水密和退化几何等证据。
4. 增加拆件目标、FDM / 光固化 / SLS 和多项优先目标输入。
5. 本地 Mock 生成完整结构化结果：是否需要拆件、原因、建议件数、候选方案、优缺点、风险和下一步。
6. 候选方案支持点击比较；阶段二切割预览明确禁用。
7. 场景或打印床变化后旧结果立即标为过期，避免拿旧分析指导新模型。

## 产品与安全边界

- 本版没有调用第三方大模型 API，界面持续显示“本地 Mock”。
- 分析只读，不修改场景、不生成零件、不写操作历史。
- 当前没有薄壁、局部过悬、真实切面候选和多视角截图能力，界面如实显示“未检测 / 不可用”。
- 现有 `floating` 只表示整件离床，不能冒充局部过悬分析。
- 非水密或退化几何优先建议修复，不输出伪精确切面。
- `OPENAI_API_KEY` 尚未进入本版运行链路；下一批接入时只能放在后端环境变量或 Secret。

## 部署后手测

1. 打开示例或导入模型，确认右栏出现“AI 拆件”，并持续显示“只读 / 不会修改模型 / 本地 Mock”。
2. 运行打印检查后进入 AI 拆件，确认对象数、打印床和检查状态来自当前场景。
3. 编辑目标，切换打印方式和优先目标，生成建议。
4. 检查结果字段完整，候选方案可切换，阶段二预览按钮不可用。
5. 将打印床从 256 改为 180，确认旧检查和旧分析立即显示已过期。
6. 进入历史 Tab，确认分析操作没有新增历史记录。
