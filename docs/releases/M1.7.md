## 为什么先做这个版本

打印检查此前只能指出非水密和退化面，用户仍要离开产品寻找外部工具；AI 拆件建议也只能告诉用户“先修模型”，没有可执行出口。M1.7 先把高频且可验证的修复动作做成产品闭环，再继续切割预览和 Agent 工具化。

## 产品体验变化

1. 非水密或退化面错误旁新增“修复预览”，不再只有文字告警。
2. 修复计算运行在独立浏览器 Worker 中，不占用 API、不消耗生成配额，也不阻塞 3D 画布。
3. 预览卡显示面片、开放边和退化面修复前后对比，并列出实际执行动作。
4. 视口用绿色半透明层显示修复后网格；新增封口面使用更亮的绿色标记。
5. 预览阶段不改变场景、不新增历史；场景变化会令预览失效。
6. 用户确认“生成修复副本”后才创建派生资产并切换当前实例；原始资产始终保留。
7. 生成副本是单条“修复”历史，Ctrl+Z 同时恢复原实例引用并移除派生资产，重做可恢复。
8. 确认后自动重新运行打印检查，验证非水密和退化问题是否消失。

## 当前确定性修复范围

- 按与打印检查一致的容差焊接重复或近重顶点。
- 删除顶点塌缩、零面积三角形和完全重复面。
- 只封闭方向一致、最多 128 个顶点且近似共面的简单边界环。
- 修复结果必须重新通过水密、非流形边和非零封闭体积验证，否则丢弃结果。
- 检测到非流形边、开放链、边界分叉、绕序冲突、非平面孔洞或不稳定体积时拒绝自动处理。

## 明确不承诺的能力

- 不是通用 MeshLab/Netfabb 替代品，不重建复杂破损拓扑。
- 不修复任意非流形结构、非平面大洞、内部壳、布尔残片或自交。
- 当前拓扑检查不能证明模型“无自交”；水密只表示每条边具有正确邻接数。
- 封口可能改变本来有意保留的开口，因此必须先预览并由用户确认。
- Agent 仍是只读建议者，本版修复由用户从确定性检查结果中主动发起。

## 自动化覆盖

- 简单平面开口盒：10 面、4 条开放边 → 12 面、0 开放边、水密。
- 封闭模型附加退化面：删除退化面后保持水密。
- 三面共边的非流形结构：fail-closed，不产出修复几何。
- 非平面开口：fail-closed，不产出修复几何。
- 派生资产：原资产保留，撤销/重做原子恢复。
- 产品闭环：预览零历史 → 用户确认 → 派生资产 → 自动复检 → 撤销恢复。

## 手动验收清单

1. 打开示例场景，点击“打印检查”，在“开口盒 · 非水密样例”错误旁点击“修复预览”。
2. 预览卡应显示面片 `10 → 12`、开放边 `4 → 0`、退化面 `0 → 0`，并注明“原模型不变”。
3. 视口中的开口盒整体出现淡绿色预览，顶部新增两片亮绿色封口面。
4. 点击“取消预览”：场景树、资产数量和历史计数均不改变。
5. 再次预览并点击“生成修复副本”：当前实例名称带“修复版”，资产库同时保留原资产与新副本。
6. 打印检查应自动重跑；新副本不再出现“非水密网格”错误。
7. 打开历史，最后一条应为“生成网格修复副本”；Ctrl+Z 恢复原开口盒，重做恢复修复版。
8. 对非流形或复杂非平面破损模型点击修复预览，应明确说明拒绝原因，不生成可确认结果。

## 下一批建议

1. 增加自交、内部壳和孤立碎片的只读检测，先补证据能力再决定修复范围。
2. 增加修复前后局部差异视图和“仅显示新增/删除面”。
3. 把成熟的确定性修复封装为 `repair_mesh` 工具，但仍保持“预览 → 用户确认 → 执行”。
4. 再进入平面切割候选与只读切割预览；不直接开放自动拆件。
