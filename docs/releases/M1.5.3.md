## 产品变化

1. 原“图生即将支持”占位改为“文字 / 单图 / 多图”三个可操作入口。
2. 单图模式提供大面积点击与拖拽区；选择后展示预览，并支持替换、删除。
3. 多图模式提供三个固定视角：正面（必填）、左侧、右侧；至少两张，最多三张。
4. 从正面槽打开文件选择器时可一次选多张，按正面 → 左侧 → 右侧自动填充。
5. 图片模式会把底部工作区扩展到 210px；默认桌面和约 1366px 宽度均已完成视觉验收。

## 链路实现

```text
本地图片
  → /api/generate multipart
  → Worker 双重校验
  → Turnstile / 配额
  → Tripo /upload/sts
  → image_token
  → image_to_model 或 multiview_to_model
  → 既有任务轮询
  → 结果代理
  → 资产入库与场景落入
```

- 浏览器不接触服务端 Tripo key。
- 单图和多图不新增第二套任务状态机。
- 多图按 Tripo 四槽协议提交 `[front,left,back,right]`，当前产品三槽对应 `[front,left,{},right]`。
- 图片上传或任务提交失败时，已扣配额继续按 AI-07 幂等返还。

## 输入边界

- 格式：PNG、JPG/JPEG、WebP。
- 单张大小：不超过 10MB。
- 单图：恰好 1 张。
- 多图：2–3 张，正面必填。
- 本地图片二进制不持久化；刷新后可恢复已提交任务，但重试/调整前需重新选择图片。

## 上线前手测

1. 单图添加 PNG → 预览 → 删除 → 重新添加 → 生成。
2. 多图只加正面时按钮保持禁用；增加左侧或右侧后允许生成。
3. 一次选择三张图，确认按正面/左侧/右侧填充。
4. 用真实 Tripo key 分别跑通一次 `image_to_model` 和一次 `multiview_to_model`。
5. 接受结果后确认资产入库、落场、沉底、聚焦和首次打印检查仍是一条历史。
