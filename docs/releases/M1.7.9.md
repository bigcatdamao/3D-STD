## 为什么做这个版本

M1.7.8 已经能生成沿模型表面移动的真实 A/B 封口临时网格，但产品入口仍被藏在“打印检查 → 尺寸信息 → 超床”里；与之相反，右栏“AI 拆件”占据一级页签，却主要返回文字建议。用户看不出真正的拆件按钮在哪里，也无法形成稳定的主动操作路径。

M1.7.9 调整产品优先级：先让确定性功能可发现、可操作、可验证。文字型 AI 拆件页从当前 UI 撤下，拆件成为画布工具栏的一等动作。

## 新的用户操作路径

1. 在画布或场景树选中一个对象。
2. 点击画布左上“✂ 拆件”。
3. 右侧自动进入打印检查；若检查缺失或过期，系统先检查，完成后自动展开拆件工作台。
4. 在“1 选对象 → 2 调引导 → 3 看切割”中选择 X / Y / Z 方向并调整引导位置。
5. 可选扫描几何低风险位置，再调整表面吸附范围。
6. 点击“生成真实表面切割预览”，查看 A/B 临时网格、闭合接缝、尺寸和开放边验证。

没有选择时提示先选一个对象；多选时明确说明一次只处理一个对象。入口不会因为对象没有超床错误而消失。

## UI 与状态改动

- `Viewport.Toolbar` 固定增加 `data-testid="split-tool-entry"` 的“✂ 拆件”按钮；预览开启后使用绿色状态反馈。
- Inspector 从“属性 / 打印检查 / AI 拆件 / 历史”收口为“属性 / 打印检查 / 历史”。
- 打印检查底部删除“让 AI 帮你判断是否需要拆件”CTA。
- `WorkspaceLayout` 不再接受 `split` 作为当前页签；旧 localStorage 偏好读取时自动迁移到 `check`，避免升级后空白或崩溃。
- 拆件工作台在当前 PreviewStore 与尺寸 issue 匹配时可挂到任意单对象尺寸条目，不再把“超出打印床”作为主动拆件的前置条件。
- M1.6 的 API、Schema、Provider 和评测代码未删除，但当前 UI 不装载，避免文字建议继续占据核心操作界面。

## 几何能力与安全边界

- 本版本没有重写 M1.7.8 几何内核：轴向位置仍只是搜索引导，最终接缝可在带宽内沿现有网格边移动。
- 成功结果仍是 PreviewStore 中的两份真实闭合临时网格，只有 A/B 开放边都为 0 才进入 ready。
- 仍只支持单一连通、水密、边流形、无退化面的三角网格和一个无分叉闭合接缝。
- 仍不创建场景节点、不写派生资产、不写历史、不替换原模型、不导出 A/B STL。
- 中心扇形封口只证明拓扑闭合，不证明凹环/非平面环无自交，也没有结构强度、装配公差、属性继承或最终可制造验证。
