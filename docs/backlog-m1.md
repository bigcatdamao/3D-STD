## 排期原则

1. **Command 层先于一切编辑功能**:T2 是唯一单点依赖,未定型前 T5–T9 全部禁止开工——后补 command 架构等于每个功能重写一遍入栈逻辑。
2. **三泳道并行**:编辑(中)、资产/检查(左)、服务层/AI(右)互不依赖,按精力交错推进,避免单轨疲劳。
3. **mock 引擎先行**:T4 的 mock 实现让 AI 前端(T12)不等 Tripo 联通即可开发,免费 credits 全部留给联调与演示录制(技术方案 §8)。
4. **每任务有验收口径**:DoD 直接引用 PRD 验收标准,完成 = 对应样例可演示。
