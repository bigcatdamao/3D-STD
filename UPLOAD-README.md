# M1.5.3 图生模型增量上传说明

本目录以 **M1.5.2 已上传完成** 为前提，只覆盖本轮图生模型相关文件。

## 上传方式

把上传包中的文件按原有相对路径覆盖到仓库根目录：

- `src/...` → 仓库 `src/...`
- `tests/...` → 仓库 `tests/...`
- `docs/...` → 仓库 `docs/...`
- `README.md`、`UPLOAD-README.md` → 仓库根目录

不要删除仓库中的其他文件。本包不包含 `node_modules`、`dist` 或测试截图，线上构建应由仓库源码重新执行。

## 本批内容

1. AI 底栏增加“文字 / 单图 / 多图”三个生成入口。
2. 单图提供大面积点击、拖拽、预览、替换和删除。
3. 多图提供正面、左侧、右侧三个槽位；正面必填，至少两张、最多三张。
4. 前端使用 `multipart/form-data` 提交，Worker 完成格式、大小、数量和视角校验后再进入验证码与配额流程。
5. Worker 把图片上传到 Tripo 获取 `image_token`，分别提交 `image_to_model` 或 `multiview_to_model`。
6. 图片生成继续复用原有排队、轮询、返还、结果代理、资产入库、落场和打印检查链路。
7. 增加 M1.5.3 永久版本记录、图片协议测试与响应式视觉验收记录。

## 本地验证结果

- 自动化测试：`374 passed`
- TypeScript：通过
- 生产构建：通过
- 外部 Chrome：文字、单图、三图空态和约 1366px 响应式布局通过，无控制台错误

真实 Tripo 单图与多图各一次仍需部署后人工点测；这两次会消耗服务额度。

完整版本记录见 `docs/releases/M1.5.3.md`。
